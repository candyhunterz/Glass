    ```zsh
    # Glass Shell Integration for Zsh
    #
    # Emits OSC 133 (command lifecycle) and OSC 7 (CWD) sequences.
    # Compatible with Starship, Oh My Posh, and Powerlevel10k.
    #
    # Source this from ~/.zshrc:
    #   source /path/to/glass/shell-integration/glass.zsh

    [[ -n "$__GLASS_INTEGRATION_LOADED" ]] && return
    __GLASS_INTEGRATION_LOADED=1

    __glass_osc7() {
        printf '\e]7;file://%s%s\e\\' "${HOST}" "${PWD}"
    }

    __glass_precmd() {
        local exit_code=$?
        printf '\e]133;D;%d\e\\' "$exit_code"
        __glass_osc7
        printf '\e]133;A\e\\'
    }

    __glass_preexec() {
        printf '\e]133;B\e\\'
        printf '\e]133;C\e\\'
    }

    autoload -Uz add-zsh-hook
    add-zsh-hook precmd __glass_precmd
    add-zsh-hook preexec __glass_preexec
    ```

    Read the existing `shell-integration/glass.bash` first to understand the exact OSC sequence format and ensure consistency. The zsh version should emit identical OSC sequences.

    Key differences from bash:
    - Uses `add-zsh-hook` instead of PROMPT_COMMAND/DEBUG trap
    - Uses `$HOST` instead of `$HOSTNAME` (zsh convention)
    - Uses preexec instead of DEBUG trap (zsh native hook)
    - Guard variable prevents double-loading
