    Create `crates/glass_terminal/src/input.rs` implementing the keyboard encoding from Pattern 4 in 02-RESEARCH.md.

    Public API:
    ```rust
    pub fn encode_key(key: &Key, modifiers: ModifiersState, mode: TermMode) -> Option<Vec<u8>>
    ```

    Implementation (reference research Pattern 4 for full details):

    1. **Intercept cases that return None** (handled by main.rs, not forwarded to PTY):
       - Ctrl+Shift+C (clipboard copy)
       - Ctrl+Shift+V (clipboard paste)
       - Shift+PageUp (scrollback up)
       - Shift+PageDown (scrollback down)
       Return None for these so main.rs knows to handle them specially.

    2. **Ctrl+letter:** `(ch.to_ascii_lowercase() as u8) & 0x1f` — produces control characters 0x01-0x1a.

    3. **Ctrl+special:** `[` -> ESC (0x1b), `\\` -> FS (0x1c), `]` -> GS (0x1d), etc.

    4. **Alt+key:** ESC prefix (0x1b) then the character bytes.

    5. **Named keys:** Arrow keys, Home/End, PageUp/Down, Insert/Delete, F1-F12, Enter, Tab, Backspace, Escape. All with modifier parameter encoding.

    6. **Arrow key mode awareness:** Check `mode.contains(TermMode::APP_CURSOR)`. In app cursor mode with no modifiers, arrows use SS3 (ESC O A). In normal mode, use CSI (ESC [ A). With modifiers, always CSI with modifier param.

    7. **Modifier param calculation:** `modifier_code(mods)` function: Shift=1, Alt=2, Ctrl=4. Result = 1 + bitmask if any modifier, else 0.

    8. **Function keys:** F1-F4 use SS3 (ESC O P/Q/R/S) without modifiers, CSI with modifiers. F5-F12 use CSI with tilde codes (15,17,18,19,20,21,23,24).

    9. **Plain character:** Forward `key.to_text()` bytes if no relevant modifier.

    Update `crates/glass_terminal/src/lib.rs`:
    - Add `pub mod input;`
    - Add `pub use input::encode_key;`

  <action>
    Replace the ASCII-only keyboard handler in main.rs with the full input encoder and add clipboard + scrollback.

    **1. Replace KeyboardInput handler:**

    Change the `WindowEvent::KeyboardInput { event, .. }` handler from:
    ```rust
    if let Some(text) = event.text {
        let _ = ctx.pty_sender.send(PtyMsg::Input(Cow::Owned(text.as_bytes().to_vec())));
    }
    ```
    To:
    ```rust
    if event.state == ElementState::Pressed {
        let modifiers = /* current modifier state */;
        let mode = ctx.term.lock().mode();

        // Check for Glass-handled keys first
        if modifiers.control_key() && modifiers.shift_key() {
            match &event.logical_key {
                Key::Character(c) if c.as_str() == "c" || c.as_str() == "C" => {
                    // Ctrl+Shift+C: copy selection to clipboard
                    clipboard_copy(&ctx.term);
                    return;
                }
                Key::Character(c) if c.as_str() == "v" || c.as_str() == "V" => {
                    // Ctrl+Shift+V: paste from clipboard
                    clipboard_paste(&ctx.pty_sender, mode);
                    return;
                }
                _ => {}
            }
        }
        if modifiers.shift_key() && !modifiers.control_key() && !modifiers.alt_key() {
            match &event.logical_key {
                Key::Named(NamedKey::PageUp) => {
                    ctx.term.lock().scroll_display(Scroll::PageUp);
                    ctx.window.request_redraw();
                    return;
                }
                Key::Named(NamedKey::PageDown) => {
                    ctx.term.lock().scroll_display(Scroll::PageDown);
                    ctx.window.request_redraw();
                    return;
                }
                _ => {}
            }
        }

        // Forward to PTY via encoder
        if let Some(bytes) = encode_key(&event.logical_key, modifiers, mode) {
            let _ = ctx.pty_sender.send(PtyMsg::Input(Cow::Owned(bytes)));
        }
    }
    ```

    **CRITICAL:** Must track modifier state. winit 0.30.13 provides modifiers via `WindowEvent::ModifiersChanged` event. Add a `modifiers: ModifiersState` field to `Processor` or `WindowContext`, update it on `ModifiersChanged`, and use it in `KeyboardInput`.

    **2. Add MouseWheel handler:**

    Add `WindowEvent::MouseWheel { delta, .. }` handler:
    - Convert delta to line count (for `LineDelta(_, y)`: use y as lines; for `PixelDelta`: divide by cell_height)
    - Call `ctx.term.lock().scroll_display(Scroll::Delta(lines as i32))`
    - Request redraw

    **3. Add scrollback-reset-on-new-output:**

    In the `AppEvent::TerminalDirty` handler, check if the terminal is scrolled up. If so, reset to bottom:
    ```rust
    {
        let mut term = ctx.term.lock();
        if term.grid().display_offset() > 0 {
            term.scroll_display(Scroll::Bottom);
        }
    }
    ```
    NOTE: This auto-scrolls to bottom on new output. Some users prefer to stay scrolled — but for Phase 2, auto-scroll is the correct default behavior matching other terminals.

    Actually, on reflection: most terminals do NOT auto-scroll on new output if the user has scrolled up. They only auto-scroll when the cursor moves (i.e., new content is written). alacritty_terminal handles this internally — `Scroll::Bottom` is called when the program writes to the active screen. So the TerminalDirty handler should just request_redraw without forcing scroll. The scroll position is managed by alacritty_terminal itself. Remove the auto-scroll logic and just request redraw.

    **4. Clipboard helper functions:**

    Add at module level or in a local `clipboard` module:
    ```rust
    fn clipboard_copy(term: &Arc<FairMutex<Term<EventProxy>>>) {
        let term = term.lock();
        if let Some(selection) = term.selection_to_string() {
            if let Ok(mut clipboard) = arboard::Clipboard::new() {
                let _ = clipboard.set_text(selection);
            }
        }
    }

    fn clipboard_paste(sender: &EventLoopSender, mode: TermMode) {
        if let Ok(mut clipboard) = arboard::Clipboard::new() {
            if let Ok(text) = clipboard.get_text() {
                let bytes = if mode.contains(TermMode::BRACKETED_PASTE) {
                    let mut buf = Vec::new();
                    buf.extend_from_slice(b"\x1b[200~");
                    buf.extend_from_slice(text.as_bytes());
                    buf.extend_from_slice(b"\x1b[201~");
                    buf
                } else {
                    text.into_bytes()
                };
                let _ = sender.send(PtyMsg::Input(Cow::Owned(bytes)));
            }
        }
    }
    ```

    NOTE: `term.selection_to_string()` may or may not exist in alacritty_terminal 0.25.1. Check the actual API. If it doesn't exist, the selection extraction needs to be done manually from `term.selection()` range + grid cell access. For Phase 2, if mouse-based selection is not yet implemented, copy might only work with keyboard-selected text (vi mode). Document this limitation if mouse selection is not wired.

    **5. Add arboard dependency to root Cargo.toml:**
    ```toml
    arboard.workspace = true
    ```

    use winit::keyboard::{Key, NamedKey};
    use alacritty_terminal::grid::Scroll;
    use arboard;
