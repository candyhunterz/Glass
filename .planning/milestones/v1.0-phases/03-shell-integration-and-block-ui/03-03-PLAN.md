  <name>Task 1: Create PowerShell integration script</name>
  <files>shell-integration/glass.ps1</files>
  <action>
    Create the shell-integration/ directory and glass.ps1 based on the Windows Terminal reference pattern from RESEARCH.md.

    The script must:

    1. **Track command history ID** to detect new commands vs empty prompts:
       - `$Global:__GlassLastHistoryId = -1`

    2. **Exit code detection** (Pitfall 4 from research):
       - `__Glass-Get-LastExitCode` function that checks `$?` first, then examines `$Error[0].InvocationInfo.HistoryId` against last history entry to distinguish PowerShell cmdlet errors (-1) from external program errors (`$LASTEXITCODE`)

    3. **Stash existing prompt** for Oh My Posh / Starship compatibility (Pitfall 3):
       - Before defining new prompt: `$Global:__GlassOriginalPrompt = $function:Prompt`
       - New prompt function calls the stashed original if it exists

    4. **Prompt function** that emits:
       - OSC 133;D with exit code (ending previous command, skip on first prompt)
       - OSC 7 with CWD as file://HOSTNAME/path (backslashes converted to forward slashes)
       - OSC 133;A (prompt start)
       - Original prompt output (or default "PS path> ")
       - OSC 133;B (command input start)
       - Updates `$Global:__GlassLastHistoryId`

    5. **PSReadLine Enter key handler** for OSC 133;C:
       - Only if PSReadLine module is loaded: `Set-PSReadLineKeyHandler -Key Enter`
       - Calls `[Microsoft.PowerShell.PSConsoleReadLine]::AcceptLine()` then writes 133;C
       - This marks the transition from user input to command execution

    Use PowerShell escape syntax: `` `e `` for ESC (\x1b), `` `a `` for BEL (\x07).

    Add a header comment block explaining: what the script does, how to source it (`. $env:USERPROFILE\.glass\shell-integration\glass.ps1`), and that it is compatible with Oh My Posh and Starship.
  </action>

  </verify>

<task type="auto">
  <name>Task 2: Create Bash integration script</name>
  <files>shell-integration/glass.bash</files>
  <action>
    Create glass.bash based on the RESEARCH.md reference pattern.

    The script must:

    1. **CWD reporting function** `__glass_osc7`:
       - Emits `\e]7;file://HOSTNAME/PATH\e\\` using printf
       - Uses `$HOSTNAME` and `$PWD`

    2. **PROMPT_COMMAND function** `__glass_prompt_command`:
       - Captures `$?` as exit_code FIRST (before any other command changes it)
       - Emits OSC 133;D with exit code (ending previous command)
       - Calls `__glass_osc7` for CWD
       - Sets PS1 to: `\[\e]133;A\e\\\]` + original PS1 + `\[\e]133;B\e\\\]`
       - The `\[...\]` wrapping tells bash these are non-printing characters (prevents prompt length miscalculation)

    3. **Stash original PS1** before modifying:
       - `__GLASS_ORIGINAL_PS1="$PS1"` for compatibility with Starship/other prompt tools

    4. **PROMPT_COMMAND chaining**:
       - `PROMPT_COMMAND="__glass_prompt_command${PROMPT_COMMAND:+;$PROMPT_COMMAND}"`
       - Prepends Glass function, preserves existing PROMPT_COMMAND

    5. **PS0 for OSC 133;C** (bash >= 4.4 only):
       - Check bash version: `BASH_VERSINFO[0] >= 5` or `(== 4 && BASH_VERSINFO[1] >= 4)`
       - If supported: `PS0='\[\e]133;C\e\\\]'`
       - PS0 is evaluated after command input, before execution -- perfect for 133;C

    Use bash escape syntax: `\e` for ESC, `\e\\` for ST terminator (ESC + backslash).

    Add a header comment block explaining: what the script does, how to source it (`source ~/.glass/shell-integration/glass.bash`), bash version requirement (>= 4.4 for PS0/133;C), and prompt customizer compatibility.
  </action>

  </verify>
