     - Build per-file JSON: `{ "path": path, "status": "modified"|"created"|"deleted"|"unchanged"|"binary", "diff": unified_string }`

   ```rust
   fn truncate_to_budget(text: &str, char_budget: usize) -> String {
       if text.len() <= char_budget {
           return text.to_string();
       }
       let truncated: String = text.chars().take(char_budget.saturating_sub(4)).collect();
       format!("{}...", truncated)
   }
   ```

3. Add the tool handler inside `#[tool_router] impl GlassServer` block:
   ```rust
   #[tool(description = "Get a compressed context summary of terminal activity within a token budget. Focus on specific aspects (errors, files, history) or get a balanced overview. Uses approximate token counting (1 token ~ 4 chars).")]
   async fn glass_compressed_context(
       &self,
       Parameters(params): Parameters<CompressedContextParams>,
   ) -> Result<CallToolResult, McpError>
   ```
   Implementation (all inside `tokio::task::spawn_blocking`):
   - Parse `after` time filter using `glass_history::query::parse_time` (default to "1h" if None)
   - Open HistoryDb, call `build_context_summary(conn, after_epoch)`
   - Calculate `char_budget = params.token_budget * 4`
   - Build summary header (always included): "## Context Summary\nCommands: {N}, Failed: {N}, Dirs: {N}\n"
   - Track `remaining` chars after summary header
   - Based on focus mode:
     - `"errors"`: Query failed commands (`exit_code != 0`) using `filtered_query` with appropriate filter. Format each as "- [{exit_code}] `{command}` in {cwd}\n  Output: {output_preview_truncated}\n". Truncate to remaining budget.
     - `"files"`: Open SnapshotStore, get recent snapshots. For each, list files with change type. Truncate to remaining budget.
     - `"history"`: Query recent commands using `filtered_query`. Format as "- `{command}` [{exit_code}] ({duration}ms) in {cwd}\n". Truncate to remaining budget.
     - `None` (balanced): Split remaining budget into thirds. Include a bit of each section (errors first if any, then history, then files).
   - Apply `truncate_to_budget` to final output
   - Return as `Content::text` (not JSON — this is meant to be a human/agent-readable compressed summary)
