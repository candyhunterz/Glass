```rust
#[tool(
    description = "Check if the Glass GUI process is running and responsive..."
)]
async fn glass_ping(&self) -> Result<CallToolResult, McpError> {
    let client = match self.ipc_client.as_ref() {
        Some(c) => c,
        None => {
            return Ok(CallToolResult::error(vec![Content::text(
                "Glass GUI is not running. Live tools require a running Glass window.",
            )]));
        }
    };
    match client.send_request("ping", serde_json::json!({})).await {
        Ok(resp) => Ok(CallToolResult::success(vec![Content::text(
            serde_json::to_string_pretty(&resp).unwrap_or_default(),
        )])),
        Err(e) => Ok(CallToolResult::error(vec![Content::text(format!(
            "Failed to communicate with Glass GUI: {}", e
        ))])),
    }
}
```

1. Add parameter types after the existing param types section (before GlassServer struct). Use the research-recommended approach with separate optional fields (NOT serde flatten, since schemars compatibility is uncertain):

   ```rust
   /// Parameters for glass_tab_create.
   #[derive(Debug, Deserialize, schemars::JsonSchema)]
   pub struct TabCreateParams {
       /// Shell to use (e.g. "bash", "pwsh"). Uses default if omitted.
       #[schemars(description = "Shell to use (e.g. 'bash', 'pwsh'). Uses default if omitted")]
       pub shell: Option<String>,
       /// Working directory for the new tab.
       #[schemars(description = "Working directory for the new tab")]
       pub cwd: Option<String>,
   }

   /// Parameters for glass_tab_send.
   #[derive(Debug, Deserialize, schemars::JsonSchema)]
   pub struct TabSendParams {
       /// 0-based tab index.
       #[schemars(description = "0-based tab index (provide this OR session_id)")]
       pub tab_index: Option<u64>,
       /// Stable session ID.
       #[schemars(description = "Stable session ID (provide this OR tab_index)")]
       pub session_id: Option<u64>,
       /// Command string to send (Enter is appended automatically).
       #[schemars(description = "Command string to send (Enter is appended automatically)")]
       pub command: String,
   }

   /// Parameters for glass_tab_output.
   #[derive(Debug, Deserialize, schemars::JsonSchema)]
   pub struct TabOutputParams {
       /// 0-based tab index.
       #[schemars(description = "0-based tab index (provide this OR session_id)")]
       pub tab_index: Option<u64>,
       /// Stable session ID.
       #[schemars(description = "Stable session ID (provide this OR tab_index)")]
       pub session_id: Option<u64>,
       /// Number of lines to return from the end of output (default 50).
       #[schemars(description = "Number of lines to return from the end (default 50, max 10000)")]
       pub lines: Option<usize>,
       /// Regex pattern to filter lines.
       #[schemars(description = "Regex pattern to filter output lines")]
       pub pattern: Option<String>,
   }

   /// Parameters for glass_tab_close.
   #[derive(Debug, Deserialize, schemars::JsonSchema)]
   pub struct TabCloseParams {
       /// 0-based tab index.
       #[schemars(description = "0-based tab index (provide this OR session_id)")]
       pub tab_index: Option<u64>,
       /// Stable session ID.
       #[schemars(description = "Stable session ID (provide this OR tab_index)")]
       pub session_id: Option<u64>,
   }
   ```

   NOTE: Inline tab_index/session_id in each struct rather than using flatten. This avoids the schemars flatten uncertainty flagged in research.

2. Add 5 `#[tool]` handler methods inside the `#[tool_router] impl GlassServer` block, right after `glass_ping`. Each follows the exact same pattern as `glass_ping`:

   **glass_tab_list** (no params):
   - Description: "List all open tabs with their state: name, working directory, session ID, and whether a command is running."
   - Send IPC request "tab_list" with empty params `{}`.
   - Return the pretty-printed JSON response.

   **glass_tab_create** (TabCreateParams):
   - Description: "Create a new terminal tab with optional shell and working directory. Returns the new tab's index and session ID."
   - Build params JSON from TabCreateParams fields (only include non-None values).
   - Send IPC request "tab_create".

   **glass_tab_send** (TabSendParams):
   - Description: "Send a command to a specific tab's terminal. The command is executed immediately (Enter is appended). Use tab_output to read results later."
   - Build params JSON including tab_index/session_id and command.
   - Send IPC request "tab_send".

   **glass_tab_output** (TabOutputParams):
   - Description: "Read the last N lines of output from a specific tab. Optionally filter lines by regex pattern. Default 50 lines, max 10000."
   - Build params JSON including tab_index/session_id, lines, pattern.
   - Send IPC request "tab_output".

   **glass_tab_close** (TabCloseParams):
   - Description: "Close a specific tab. Refuses to close the last remaining tab. Use session_id for stability (tab indices shift on close)."
   - Build params JSON including tab_index/session_id.
   - Send IPC request "tab_close".

   For building params JSON, use `serde_json::json!` and conditionally insert fields:
   ```rust
   let mut params = serde_json::json!({});
   if let Some(idx) = input.tab_index { params["tab_index"] = serde_json::json!(idx); }
   if let Some(sid) = input.session_id { params["session_id"] = serde_json::json!(sid); }
   ```

   NOTE: Each tool method takes `&self, #[tool(aggr)] input: ParamType` for parameterized tools, or just `&self` for glass_tab_list (no params). Check how existing parameterized tools like `glass_history` declare their params -- they use `#[tool(aggr)] input: HistoryParams`.
