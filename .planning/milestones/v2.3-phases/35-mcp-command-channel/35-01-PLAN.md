  <action>
    Create `crates/glass_core/src/ipc.rs` with:

    1. **Types** (serde Serialize/Deserialize):
       - `McpRequest { id: u64, method: String, params: serde_json::Value }` -- Deserialize
       - `McpResponse { id: u64, result: Option<serde_json::Value>, error: Option<String> }` -- Serialize
       - `McpEventRequest { request: McpRequest, reply: tokio::sync::oneshot::Sender<McpResponse> }` -- internal, not serde

    2. **Socket path helpers** (cfg-gated):
       - `#[cfg(unix)] pub fn ipc_socket_path() -> PathBuf` -- returns `~/.glass/glass.sock`
       - `#[cfg(windows)] pub fn ipc_pipe_name() -> String` -- returns `\\.\pipe\glass-terminal`

    3. **Connection handler** (generic over AsyncRead+AsyncWrite):
       - `async fn handle_ipc_connection<S>(stream: S, proxy: EventLoopProxy<AppEvent>)` where S: AsyncRead + AsyncWrite + Unpin
       - Reads newline-delimited JSON lines from stream
       - For each line: parse McpRequest, create oneshot channel, send AppEvent::McpRequest via proxy, await oneshot response, write JSON response line back
       - On invalid JSON: write error response, continue
       - On proxy.send_event error: break (event loop closed)

    4. **Platform-specific listeners**:
       - `#[cfg(unix)] pub async fn spawn_ipc_listener(proxy: EventLoopProxy<AppEvent>) -> anyhow::Result<()>` -- binds UnixListener on ipc_socket_path(), cleans stale socket first, loops accepting connections, spawns handle_ipc_connection per connection
       - `#[cfg(windows)] pub async fn spawn_ipc_listener(proxy: EventLoopProxy<AppEvent>) -> anyhow::Result<()>` -- creates named pipe ServerOptions, loops: create pipe instance, connect().await, spawn handler, create next pipe instance

    5. **Launcher function**:
       - `pub fn start_ipc_listener(proxy: EventLoopProxy<AppEvent>)` -- spawns a std::thread with a new tokio::runtime::Runtime (following the pattern from research open question 3), runs spawn_ipc_listener on it. Thread named "Glass IPC listener". Log info on start, warn on error.

    Update `crates/glass_core/src/event.rs`:
    - Add `McpRequest(crate::ipc::McpEventRequest)` variant to `AppEvent` enum

    Update `crates/glass_core/src/lib.rs`:
    - Add `pub mod ipc;`

    Update `crates/glass_core/Cargo.toml`:
    - Add `tokio = { workspace = true }` to dependencies (needed for oneshot, net, io, runtime)
    - Add `anyhow = { workspace = true }` to dependencies

    Note: The connection handler should include a 5-second timeout per request using `tokio::time::timeout` around the oneshot rx.await, returning a timeout error response if exceeded.
