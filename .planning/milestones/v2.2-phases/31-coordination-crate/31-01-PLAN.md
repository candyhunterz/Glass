Create `crates/glass_coordination/src/db.rs` with:

1. Schema creation (`create_schema` as associated function on CoordinationDb):
   ```sql
   CREATE TABLE IF NOT EXISTS agents (
       id             TEXT PRIMARY KEY,
       name           TEXT NOT NULL,
       agent_type     TEXT NOT NULL,
       project        TEXT NOT NULL,
       cwd            TEXT NOT NULL,
       pid            INTEGER,
       status         TEXT NOT NULL DEFAULT 'active',
       task           TEXT,
       registered_at  INTEGER NOT NULL,
       last_heartbeat INTEGER NOT NULL
   );
   CREATE INDEX IF NOT EXISTS idx_agents_project ON agents(project);
   CREATE INDEX IF NOT EXISTS idx_agents_heartbeat ON agents(last_heartbeat);

   CREATE TABLE IF NOT EXISTS file_locks (
       path      TEXT NOT NULL,
       agent_id  TEXT NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
       reason    TEXT,
       locked_at INTEGER NOT NULL,
       PRIMARY KEY (path)
   );
   CREATE INDEX IF NOT EXISTS idx_file_locks_agent ON file_locks(agent_id);

   CREATE TABLE IF NOT EXISTS messages (
       id         INTEGER PRIMARY KEY AUTOINCREMENT,
       from_agent TEXT REFERENCES agents(id) ON DELETE SET NULL,
       to_agent   TEXT REFERENCES agents(id) ON DELETE CASCADE,
       msg_type   TEXT NOT NULL,
       content    TEXT NOT NULL,
       read       INTEGER NOT NULL DEFAULT 0,
       created_at INTEGER NOT NULL
   );
   CREATE INDEX IF NOT EXISTS idx_messages_to_agent ON messages(to_agent);
   CREATE INDEX IF NOT EXISTS idx_messages_read ON messages(read);
   ```
   NOTE on messages table: `to_agent ON DELETE CASCADE` means when recipient is deleted, their messages are cleaned up. `from_agent ON DELETE SET NULL` preserves messages from deregistered senders. For broadcasts (`to_agent IS NULL`), a separate row is inserted per recipient at broadcast time (see Plan 03).

2. Migration function using `PRAGMA user_version` (version 1 for initial schema).

3. `CoordinationDb` struct with `conn: Connection`:
   - `pub fn open(path: &Path) -> Result<Self>`: Open connection, set pragmas (WAL, synchronous=NORMAL, busy_timeout=5000, foreign_keys=ON), create schema, migrate.
   - `pub fn open_default() -> Result<Self>`: Calls `open` with `resolve_db_path()`.

4. Agent operations (all use `&mut self` for `transaction_with_behavior(Immediate)`):
   - `pub fn register(&mut self, name: &str, agent_type: &str, project: &str, cwd: &str, pid: Option<u32>) -> Result<String>`: Canonicalize `project` path using `crate::canonicalize_path()`. Generate UUID v4. INSERT INTO agents. Return the UUID string.
   - `pub fn deregister(&mut self, agent_id: &str) -> Result<bool>`: DELETE FROM agents WHERE id = ?. CASCADE handles locks. SET NULL handles messages.from_agent. Return whether a row was deleted.
   - `pub fn heartbeat(&mut self, agent_id: &str) -> Result<bool>`: UPDATE agents SET last_heartbeat = unixepoch() WHERE id = ?. Return whether a row was updated.
   - `pub fn update_status(&mut self, agent_id: &str, status: &str, task: Option<&str>) -> Result<bool>`: UPDATE agents SET status = ?, task = ?, last_heartbeat = unixepoch() WHERE id = ?. Implicitly refreshes heartbeat.
   - `pub fn list_agents(&mut self, project: &str) -> Result<Vec<AgentInfo>>`: SELECT all columns from agents WHERE project = ?, map to AgentInfo.
   - `pub fn prune_stale(&mut self, timeout_secs: i64) -> Result<Vec<String>>`: In a single IMMEDIATE transaction: find agents with last_heartbeat older than timeout, find agents with dead PIDs (using `crate::pid::is_pid_alive`). Prune by heartbeat timeout regardless of PID (PID reuse concern). Prune by dead PID even if heartbeat is recent (early detection). DELETE each stale agent (CASCADE removes locks). Return list of pruned agent IDs. Use `tracing::info!` to log each pruned agent.
