  <action>
Add file locking methods to the existing `impl CoordinationDb` block in db.rs:

1. `pub fn lock_files(&mut self, agent_id: &str, paths: &[std::path::PathBuf], reason: Option<&str>) -> Result<LockResult>`:
   - Begin IMMEDIATE transaction
   - Canonicalize all paths using `crate::canonicalize_path()`
   - For each canonical path, query `SELECT fl.path, a.name, fl.reason FROM file_locks fl JOIN agents a ON fl.agent_id = a.id WHERE fl.path = ?1 AND fl.agent_id != ?2` to find conflicts
   - If ANY conflicts exist, return `LockResult::Conflict(conflicts)` without inserting anything (all-or-nothing per COORD-05)
   - If no conflicts, INSERT OR REPLACE each lock: `INSERT OR REPLACE INTO file_locks (path, agent_id, reason, locked_at) VALUES (?1, ?2, ?3, unixepoch())`
   - Also update the agent's heartbeat in the same transaction (implicit heartbeat on activity)
   - Commit and return `LockResult::Acquired(canonical_paths)`

2. `pub fn unlock_file(&mut self, agent_id: &str, path: &std::path::Path) -> Result<bool>`:
   - Canonicalize path
   - DELETE FROM file_locks WHERE path = ?1 AND agent_id = ?2
   - Return whether a row was deleted (changes() > 0)

3. `pub fn unlock_all(&mut self, agent_id: &str) -> Result<u64>`:
   - DELETE FROM file_locks WHERE agent_id = ?1
   - Return count of deleted rows

4. `pub fn list_locks(&mut self, project: Option<&str>) -> Result<Vec<FileLock>>`:
   - If project is Some: SELECT fl.path, fl.agent_id, a.name, fl.reason, fl.locked_at FROM file_locks fl JOIN agents a ON fl.agent_id = a.id WHERE a.project = ?1
   - If project is None: same query without WHERE clause (shows all locks for GUI)
   - Map rows to FileLock structs
