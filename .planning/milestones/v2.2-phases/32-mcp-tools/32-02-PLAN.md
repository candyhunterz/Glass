pub struct LockParams {

    #[schemars(description = "Agent UUID releasing locks")]

pub struct ListLocksParams {

}

**Tool handlers:**

1. `glass_agent_lock` with description "Atomically claim advisory file locks. Returns conflicts if any file is held by another agent."
   - Clone coord_db_path, spawn_blocking
   - Convert `params.paths.iter().map(PathBuf::from).collect::<Vec<_>>()`
   - Call `db.lock_files(&params.agent_id, &paths, params.reason.as_deref())`
   - Match on LockResult::Acquired(paths) -> JSON `{ locked: paths, conflicts: [] }`
   - Match on LockResult::Conflict(conflicts) -> JSON `{ locked: [], conflicts: [{path, held_by, held_by_id, reason, retry_hint: "Wait and retry, or send a 'request_unlock' message to the holder"}] }`
   - CRITICAL: Both Acquired and Conflict return CallToolResult::success -- conflicts are NOT errors

2. `glass_agent_unlock` with description "Release file locks. Omit paths to release all locks."
   - If `params.paths` is Some, iterate and call `db.unlock_file()` for each, count successes
   - If `params.paths` is None, call `db.unlock_all()`
   - Add explicit `db.heartbeat(&params.agent_id)` call for MCP-12 compliance (unlock_file/unlock_all don't refresh heartbeat internally)
   - Return JSON `{ released: count }`

3. `glass_agent_locks` with description "List all active file locks, optionally filtered by project."
   - Call `db.list_locks(params.project.as_deref())`
   - Serialize FileLock structs via serde (they derive Serialize)
   - Return JSON `{ locks: [...] }`
