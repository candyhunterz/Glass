4. **CLI Undo subcommand** (main.rs) -- UI-03:
   - Add `Undo { command_id: i64 }` variant to Commands enum (follow History/Mcp pattern)
   - In main() match arm:
     ```rust
     Some(Commands::Undo { command_id }) => {
         let cwd = std::env::current_dir().unwrap_or_default();
         let glass_dir = glass_snapshot::resolve_glass_dir(&cwd);
         match glass_snapshot::SnapshotStore::open(&glass_dir) {
             Ok(store) => {
                 let engine = glass_snapshot::UndoEngine::new(&store);
                 match engine.undo_command(command_id) {
                     Ok(Some(result)) => {
                         println!("Undo complete for command {} ({:?} confidence):", command_id, result.confidence);
                         for (path, outcome) in &result.files {
                             let status = match outcome {
                                 glass_snapshot::FileOutcome::Restored => "restored",
                                 glass_snapshot::FileOutcome::Deleted => "deleted",
                                 glass_snapshot::FileOutcome::Skipped => "skipped",
                                 glass_snapshot::FileOutcome::Conflict { .. } => "CONFLICT",
                                 glass_snapshot::FileOutcome::Error(e) => { eprintln!("  error {}: {}", path.display(), e); continue; }
                             };
                             println!("  {} {}", status, path.display());
                         }
                     }
                     Ok(None) => { eprintln!("No snapshot found for command {}", command_id); std::process::exit(1); }
                     Err(e) => { eprintln!("Undo failed: {}", e); std::process::exit(1); }
                 }
             }
             Err(e) => { eprintln!("Failed to open snapshot store: {}", e); std::process::exit(1); }
         }
         return Ok(());
     }
     ```

5. **Undo visual feedback** (main.rs) -- UI-02:
   - In the Ctrl+Shift+Z handler, after `engine.undo_latest()` returns `Ok(Some(result))`:
   - **Final decision for V1:** After undo, set `has_snapshot = false` on the block (label disappears = visual confirmation undo happened). Log detailed per-file outcomes via tracing::info! with format: "Undo: {status} {path}" for each file. This is the minimal viable feedback -- the disappearance of [undo] IS visual confirmation. Enhanced overlay can come in a future phase.
   - Additionally, log a summary line: "Undo complete: N restored, M deleted, K skipped, J conflicts"

6. **Startup pruning** (main.rs) -- STOR-01 wiring:
   - In window creation, after snapshot_store is successfully opened, spawn a background thread for pruning:
     ```rust
     if let Some(ref store) = ctx.snapshot_store {
         let glass_dir = glass_snapshot::resolve_glass_dir(&std::env::current_dir().unwrap_or_default());
         let config = self.config.snapshot.clone();
         std::thread::spawn(move || {
             let store = match glass_snapshot::SnapshotStore::open(&glass_dir) {
                 Ok(s) => s,
                 Err(e) => { tracing::warn!("Pruning: failed to open store: {}", e); return; }
             };
             let snap_config = config.unwrap_or_default();
             let pruner = glass_snapshot::Pruner::new(&store, snap_config.retention_days, snap_config.max_count, snap_config.max_size_mb);
             match pruner.prune() {
                 Ok(result) => tracing::info!("Pruning complete: {} snapshots, {} blobs removed", result.snapshots_deleted, result.blobs_deleted),
                 Err(e) => tracing::warn!("Pruning failed: {}", e),
             }
         });
     }
     ```
   - Note: Open a new SnapshotStore in the thread (SnapshotStore is not Send since it holds a rusqlite Connection). This is fine -- it opens the same DB file with WAL mode providing read isolation.
