  <name>Task 1: Add pruning DB queries + Pruner module</name>
  <files>crates/glass_snapshot/src/db.rs, crates/glass_snapshot/src/pruner.rs, crates/glass_snapshot/src/lib.rs</files>
  <behavior>
    - Test: delete_snapshots_before(epoch) removes snapshots with created_at < epoch and returns deleted IDs
    - Test: count_snapshots() returns correct count
    - Test: get_oldest_snapshot_ids(limit) returns IDs ordered by created_at ASC
    - Test: get_referenced_hashes() returns only hashes still referenced by snapshot_files
    - Test: Pruner.prune() with retention_days=0 deletes all snapshots
    - Test: Pruner.prune() with max_count=2 keeps only 2 newest snapshots when 5 exist
    - Test: Pruner.prune() cleans up orphan blob files after snapshot deletion
    - Test: Pruner.prune() skips the 10 most recent snapshots regardless of age (safety margin)
  </behavior>
  <action>
1. Add these methods to SnapshotDb in db.rs:
   - `pub fn delete_snapshots_before(&self, epoch: i64) -> Result<Vec<i64>>`: SELECT ids WHERE created_at < epoch, then DELETE, return ids
   - `pub fn count_snapshots(&self) -> Result<u64>`: SELECT COUNT(*) FROM snapshots
   - `pub fn get_oldest_snapshot_ids(&self, limit: u32) -> Result<Vec<i64>>`: SELECT id FROM snapshots ORDER BY created_at ASC LIMIT ?
   - `pub fn get_referenced_hashes(&self) -> Result<HashSet<String>>`: SELECT DISTINCT blob_hash FROM snapshot_files WHERE blob_hash IS NOT NULL

2. Create pruner.rs module:
   - `pub struct Pruner<'a> { store: &'a SnapshotStore, config: SnapshotSection }`
   - `pub fn new(store: &'a SnapshotStore, config: &SnapshotSection) -> Self`
   - `pub fn prune(&self) -> Result<PruneResult>` where PruneResult has fields: snapshots_deleted: u32, blobs_deleted: u32
   - Pruning logic order:
     a. Delete by age: compute epoch = now - retention_days * 86400, call delete_snapshots_before(epoch) but skip the 10 most recent (get total count, if count <= 10 skip age pruning; otherwise compute safe_epoch by getting the 10th newest snapshot's created_at and using max(age_epoch, safe_epoch))
     b. Delete by count: if count_snapshots() > max_count, get_oldest_snapshot_ids(count - max_count), delete each via delete_snapshot(id)
     c. Orphan blob cleanup: get_referenced_hashes(), list all blob files in blob_dir (BlobStore stores in glass_dir/blobs/XX/HASH), walk the directory, delete files whose hash is not in the referenced set
   - For blob directory walking: iterate glass_dir/blobs/ subdirectories, each file's stem is the hash. Use std::fs::read_dir recursively.

3. Add `pub fn list_blob_hashes(&self) -> Result<Vec<String>>` to BlobStore in blob_store.rs: walk the blobs/ directory, collect all hash filenames.

4. Update lib.rs: add `pub mod pruner;` and `pub use pruner::Pruner;`

5. Add SnapshotSection to glass_snapshot's dependencies if not already present (it needs glass_core for the config type). Alternative: pass individual config values (retention_days, max_count, max_size_mb) as constructor args to avoid the dependency. Prefer passing individual values to keep crate boundaries clean:
   `pub fn new(store: &'a SnapshotStore, retention_days: u32, max_count: u32, max_size_mb: u32) -> Self`

Note: max_size_mb enforcement requires summing blob file sizes. Walk blob directory, sum sizes, if over limit delete oldest snapshots iteratively until under. For V1, enforce count and age first; size enforcement is a secondary check.
