  <behavior>
    - split_pipes("cat file | grep foo | wc -l") returns ["cat file", "grep foo", "wc -l"]
    - split_pipes("echo hello") returns ["echo hello"] (no pipe = single stage)
    - split_pipes("echo 'hello | world'") returns ["echo 'hello | world'"] (pipe inside single quotes)
    - split_pipes("echo \"hello | world\"") returns ["echo \"hello | world\""] (pipe inside double quotes)
    - split_pipes("echo hello \\| world") returns ["echo hello \\| world"] (backslash-escaped pipe)
    - split_pipes("echo hello `| world") returns ["echo hello `| world"] (backtick-escaped pipe for PowerShell)
    - split_pipes("cmd1 || cmd2") returns ["cmd1 || cmd2"] (logical OR, not pipe)
    - split_pipes("cmd1 || cmd2 | cmd3") returns ["cmd1 || cmd2", "cmd3"] (OR + pipe)
    - split_pipes("echo $(cat file | grep foo) | wc") returns ["echo $(cat file | grep foo)", "wc"] (pipe inside $() is not top-level)
    - split_pipes("(cmd1 | cmd2) | cmd3") returns ["(cmd1 | cmd2)", "cmd3"] (pipe inside parens is not top-level)
    - parse_pipeline("cat file | grep foo | wc -l") returns Pipeline with 3 stages, programs ["cat", "grep", "wc"]
    - parse_pipeline("ls") returns Pipeline with 1 stage, program "ls", stages[0].index == 0
    - parse_pipeline("/usr/bin/cat file | grep foo") extracts program "cat" (path-stripped)
    - parse_pipeline("") returns Pipeline with 1 empty stage
  </behavior>

Create crates/glass_pipes/src/parser.rs with two public functions:

1. `pub fn split_pipes(command: &str) -> Vec<&str>` -- Byte-level state machine that scans for unquoted, unescaped `|` characters at parenthesis depth 0. Track state:
   - `in_single_quote: bool` (toggle on `'` when not in double quote)
   - `in_double_quote: bool` (toggle on `"` when not in single quote)
   - `escaped: bool` (set on `\` outside single quotes, OR on `` ` `` for PowerShell support)
   - `paren_depth: usize` (increment on `(` when not in quotes, decrement on `)`)
   - Also track `$(` as entering a subshell context (paren_depth handles this naturally since `$` followed by `(` will increment depth on the `(`)

   When an unquoted, unescaped `|` is found at depth 0: check next byte. If also `|`, skip both (logical OR). Otherwise, record a stage boundary. Return trimmed slices.

2. `pub fn parse_pipeline(command: &str) -> Pipeline` -- Calls split_pipes, then for each stage:
   - Use `shlex::split()` on the stage text to get tokens
   - First token is the program path; strip to basename using `rsplit` on `/` and `\`
   - Create `PipeStage { command, index, program, is_tty: false }`
   - Return `Pipeline { raw_command: command.to_string(), stages, classification: PipelineClassification::default() }`

Write the tests FIRST (RED), then implement (GREEN). Tests go in a `#[cfg(test)] mod tests` block at the bottom of parser.rs.
