<task type="auto">
  <name>Task 2: Create history.rs with display formatting and subcommand dispatch</name>
  <files>src/history.rs</files>
  <action>
1. Create `src/history.rs` with a `pub fn run_history(action: Option<crate::HistoryAction>)` function that:

   a. Resolves the database path: `let db_path = glass_history::resolve_db_path(&std::env::current_dir().unwrap_or_default());`
   b. Opens the database: `let db = glass_history::HistoryDb::open(&db_path)` -- on error, print user-friendly message to stderr and exit(1).
   c. Matches on action:
      - `None` or `Some(HistoryAction::List { filters })`: Build QueryFilter from filters (no text), call `db.filtered_query(&filter)`, print results.
      - `Some(HistoryAction::Search { query, filters })`: Build QueryFilter from filters with `text = Some(query)`, call `db.filtered_query(&filter)`, print results.
   d. For `--after` and `--before` string values, call `glass_history::query::parse_time()` to convert to epoch. On parse error, print helpful message and exit(1).

2. Add display formatting functions:
   - `fn format_timestamp(epoch: i64) -> String`: Use `chrono::DateTime::from_timestamp(epoch, 0)` to format as "YYYY-MM-DD HH:MM:SS". If timestamp is within last 24h, show relative like "2h ago", "45m ago". Otherwise show the date.
   - `fn format_duration(ms: i64) -> String`: <1000 => "Nms", <60000 => "N.Ns", else "NmNs".
   - `fn truncate(s: &str, max: usize) -> String`: Truncate with "..." if longer than max.
   - `fn print_results(records: &[CommandRecord])`: If empty, print "No matching commands found." and return. Otherwise print a header line: `"{:<50} {:>4} {:>8} {:>16} {}"` for COMMAND, EXIT, DURATION, TIME, CWD. Print a separator line of dashes. For each record, print formatted row. Show `<empty>` for blank command text. Show `-` for None exit code. Print count footer: `"\n{} result(s)"`.

3. Add `mod history;` to `src/main.rs` (if not already added in Task 1).

4. Wire the match arm in main.rs: `Some(Commands::History { action }) => { history::run_history(action); }`

Key details:
- Do NOT create a window or event loop for history subcommands. The current main.rs structure already handles this correctly (subcommands are matched before event loop creation).
- Use `std::process::exit(0)` at the end of run_history for clean exit.
- chrono is a dependency of glass_history, not the glass binary. Use glass_history's parse_time function, and for display formatting add chrono to glass binary's Cargo.toml OR format timestamps using the epoch values directly with glass_history's chrono re-export. Simplest: add `chrono = { workspace = true }` to the root Cargo.toml [dependencies].
