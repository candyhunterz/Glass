**2. Implement scroll-to-block on Enter key:**

In the overlay input interception block (from Plan 01), update the Enter handler:

```rust
Key::Named(NamedKey::Enter) => {
    if overlay.selected < overlay.results.len() {
        // Get the selected CommandRecord's started_at timestamp
        let selected_record = &overlay.results[overlay.selected];
        let target_timestamp = selected_record.started_at;

        // Find the closest matching block by timestamp.
        // Block.started_at is Instant (monotonic), so we can't directly match.
        // Instead, iterate blocks and find one whose prompt_start_line is closest
        // to a reasonable position. For v1.1, use a simple heuristic:
        // find the block at the approximate position based on block ordering.
        //
        // Since blocks are stored in chronological order and CommandRecords have
        // epoch timestamps, we match by index position: the Nth most recent
        // block corresponds to the Nth most recent command record.

        let all_blocks = ctx.block_manager.all_blocks();
        // Search results are ordered by started_at DESC (most recent first).
        // Find the block that best matches by scanning all blocks for the closest
        // started_at wall time match. Since we don't have wall time on blocks,
        // use a positional heuristic: if the result is at index I in a reverse-chronological
        // list, use the Ith block from the end.
        if !all_blocks.is_empty() {
            // Simple positional mapping: results are newest-first,
            // blocks are oldest-first. Use block from the end.
            let block_idx = all_blocks.len().saturating_sub(overlay.selected + 1);
            if block_idx < all_blocks.len() {
                let target_line = all_blocks[block_idx].prompt_start_line;
                let mut term = ctx.term.lock();
                let history_size = term.grid().history_size();
                let current_offset = term.grid().display_offset();
                // display_offset = distance from bottom of history
                // target display_offset to show target_line at top of viewport
                let target_offset = history_size.saturating_sub(target_line);
                let delta = target_offset as i32 - current_offset as i32;
                if delta != 0 {
                    term.scroll_display(alacritty_terminal::grid::Scroll::Delta(delta));
                }
            }
        }
    }
    ctx.search_overlay = None;
    ctx.window.request_redraw();
    return;
}
```

Note: This scroll-to-block approach uses positional heuristics since `Block.started_at` is `Instant` (monotonic, not wall clock) and cannot be directly compared to `CommandRecord.started_at` (epoch). The research notes this as an open question -- the simple positional mapping is the recommended v1.1 approach. If `BlockManager` does not expose `all_blocks()`, check for an equivalent method or access the blocks vec directly. If no public accessor exists, add a `pub fn all_blocks(&self) -> &[Block]` method to BlockManager.
