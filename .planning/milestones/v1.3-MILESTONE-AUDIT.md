## Cross-Phase Integration (4/4 flows)

### Flow 1: Full Pipeline Capture
User types piped command -> shell rewrites with tee/Tee-Object -> OSC 133;S/P emitted -> OscScanner parses -> pty.rs converts to ShellEvent -> main.rs processes -> BlockManager stores -> parse_pipeline extracts commands -> auto-expand triggers -> UI renders -> DB persists

**Status: COMPLETE** (11 steps wired)
**Requirements:** PIPE-01, CAPT-01, CAPT-02, CAPT-03, CAPT-04, UI-01, UI-02, UI-03, UI-04, STOR-01

### Flow 2: MCP Inspection
AI calls glass_pipe_inspect(command_id, stage) -> spawn_blocking DB query -> get_pipe_stages -> PipeStageEntry JSON response
AI calls glass_context -> SQL JOIN on pipe_stages -> pipeline_count, avg_pipeline_stages, pipeline_failure_rate

**Status: COMPLETE**
**Requirements:** MCP-01, MCP-02, STOR-01

### Flow 3: Config Disable (Three-Layer Gate)
pipes.enabled=false in config.toml -> main.rs reads config -> PTY env var GLASS_PIPES_DISABLED=1 injected -> shell scripts early-return (bash line 210, ps1 line 232) -> main.rs skips PipelineStart/PipelineStage events -> no temp file reads -> no block pipeline_stages -> no DB storage

**Status: COMPLETE** (Phase 20 closed the partial gate from prior audit)
**Requirements:** CONF-01

### Flow 4: Retention Cascade
Age/size prune -> explicit DELETE FROM pipe_stages (retention.rs) -> DELETE FROM commands -> FK ON DELETE CASCADE as safety net. delete_command also explicitly cascades. PRAGMA foreign_keys = ON.

**Status: COMPLETE** (belt-and-suspenders)
**Requirements:** STOR-02
