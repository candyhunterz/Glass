<task type="auto" tdd="true">
  <name>Task 1: Create ScrollbarRenderer with unit tests</name>
  <files>crates/glass_renderer/src/scrollbar.rs, crates/glass_renderer/src/lib.rs</files>
  <behavior>
    - build_scrollbar_rects with history_size=100, screen_lines=24, display_offset=0 returns 2 RectInstances (track + thumb), thumb near bottom of track
    - build_scrollbar_rects with display_offset=history_size returns thumb near top of track
    - build_scrollbar_rects with display_offset=history_size/2 returns thumb near middle
    - Thumb height = max(pane_height * screen_lines / (history_size + screen_lines), MIN_THUMB_HEIGHT)
    - When history_size=0, thumb fills entire track (thumb_height == pane_height)
    - When history is very large, thumb height clamps to MIN_THUMB_HEIGHT (20px)
    - is_hovered=true or is_dragging=true produces THUMB_COLOR_ACTIVE [0.588, 0.588, 0.588, 0.7]
    - is_hovered=false and is_dragging=false produces THUMB_COLOR_REST [0.392, 0.392, 0.392, 0.4]
    - Track rect always has TRACK_COLOR [1.0, 1.0, 1.0, 0.03]
    - hit_test returns None when mouse_x is outside scrollbar x-range
    - hit_test returns Some(ScrollbarHit::Thumb) when mouse is within thumb y-range
    - hit_test returns Some(ScrollbarHit::TrackAbove) when mouse is above thumb
    - hit_test returns Some(ScrollbarHit::TrackBelow) when mouse is below thumb
  </behavior>
  <action>
Create `crates/glass_renderer/src/scrollbar.rs` following the TabBarRenderer pattern:

1. Define constants:
   - `pub const SCROLLBAR_WIDTH: f32 = 8.0`
   - `const MIN_THUMB_HEIGHT: f32 = 20.0`
   - `const TRACK_COLOR: [f32; 4] = [1.0, 1.0, 1.0, 0.03]`
   - `const THUMB_COLOR_REST: [f32; 4] = [100.0/255.0, 100.0/255.0, 100.0/255.0, 0.4]`
   - `const THUMB_COLOR_ACTIVE: [f32; 4] = [150.0/255.0, 150.0/255.0, 150.0/255.0, 0.7]`

2. Define `ScrollbarHit` enum with variants: `Thumb`, `TrackAbove`, `TrackBelow`.

3. Define `ScrollbarRenderer` struct (empty or with `width: f32` field).

4. Implement `ScrollbarRenderer`:
   - `pub fn new() -> Self` — sets width to SCROLLBAR_WIDTH
   - `pub fn width(&self) -> f32` — returns the scrollbar width
   - `pub fn build_scrollbar_rects(&self, pane_right_x, pane_y, pane_height, display_offset, history_size, screen_lines, is_hovered, is_dragging) -> Vec<RectInstance>` — returns track rect + thumb rect. Scroll math: `scroll_ratio = 1.0 - (display_offset as f32 / history_size as f32)` for thumb_y positioning. When history_size==0, thumb fills entire track.
   - `pub fn hit_test(&self, mouse_x, mouse_y, scrollbar_x, viewport_y, viewport_height, display_offset, history_size, screen_lines) -> Option<ScrollbarHit>` — checks if mouse is in scrollbar region, then determines Thumb/TrackAbove/TrackBelow based on thumb y position. Reuse the same thumb geometry math as build_scrollbar_rects.
   - `pub fn compute_thumb_geometry(&self, track_height, history_size, screen_lines, display_offset) -> (f32, f32)` — returns (thumb_y_offset_within_track, thumb_height). Public so Plan 02 can use it for drag math.

5. In `lib.rs`: add `pub mod scrollbar;` and add re-export `pub use scrollbar::{ScrollbarRenderer, ScrollbarHit, SCROLLBAR_WIDTH};`

6. Write `#[cfg(test)] mod tests` inside scrollbar.rs with tests for all behaviors listed above.
  </action>
