  <action>
Create `.github/workflows/release.yml` with the following structure:

**Trigger:** `push: tags: ["v*"]`

**Permissions:** `contents: write` (needed for creating releases and uploading assets)

**Environment:** `CARGO_TERM_COLOR: always`

**Three parallel jobs:**

### Job 1: build-windows (runs-on: windows-latest)
Steps:
1. `actions/checkout@v4`
2. `dtolnay/rust-toolchain@stable`
3. `Swatinem/rust-cache@v2` with key: `release-x86_64-pc-windows-msvc`
4. **Verify version matches tag** (bash shell):
   ```bash
   CARGO_VERSION=$(grep '^version' Cargo.toml | head -1 | sed 's/.*"\(.*\)"/\1/')
   TAG_VERSION="${GITHUB_REF_NAME#v}"
   if [ "$CARGO_VERSION" != "$TAG_VERSION" ]; then
     echo "ERROR: Cargo.toml version ($CARGO_VERSION) does not match tag ($TAG_VERSION)"
     exit 1
   fi
   ```
5. `cargo build --release`
6. `cargo install cargo-wix`
7. Add WiX to PATH: `echo "C:\Program Files (x86)\WiX Toolset v3.14\bin" >> $GITHUB_PATH` (bash shell)
8. `cargo wix --no-build --nocapture`
9. `softprops/action-gh-release@v2` with `files: target/wix/*.msi`

### Job 2: build-macos (runs-on: macos-latest)
Steps:
1. `actions/checkout@v4`
2. `dtolnay/rust-toolchain@stable`
3. `Swatinem/rust-cache@v2` with key: `release-aarch64-apple-darwin`
4. **Verify version matches tag** (same script as Windows)
5. `cargo build --release`
6. `bash packaging/macos/build-dmg.sh "${GITHUB_REF_NAME#v}"` -- strip the "v" prefix so the version is just "0.1.0"
7. `softprops/action-gh-release@v2` with `files: target/macos/*.dmg`

### Job 3: build-linux (runs-on: ubuntu-latest)
Steps:
1. `actions/checkout@v4`
2. `dtolnay/rust-toolchain@stable`
3. `Swatinem/rust-cache@v2` with key: `release-x86_64-unknown-linux-gnu`
4. Install Linux dependencies: `sudo apt-get update && sudo apt-get install -y libwayland-dev libxkbcommon-dev libx11-dev libxi-dev libxtst-dev`
5. **Verify version matches tag** (same script as Windows)
6. `cargo build --release`
7. `cargo install cargo-deb`
8. `cargo deb --no-build`
9. `softprops/action-gh-release@v2` with `files: target/debian/*.deb`

**Important details:**
- All three jobs run in PARALLEL (no needs: dependencies between them)
- softprops/action-gh-release@v2 handles the race condition gracefully -- first job creates the release, others add assets
- Use `shell: bash` for the version verification step on Windows (GitHub Actions defaults to PowerShell on Windows)
- Include a release body note about macOS Gatekeeper: "macOS users: If blocked by Gatekeeper, run `xattr -cr /Applications/Glass.app` or right-click > Open."
- Use `generate_release_notes: true` on the softprops action to auto-generate changelog from commits
