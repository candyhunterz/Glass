  <implementation>
    Create `crates/glass_core/src/activity_stream.rs` with:

    1. **ActivityEvent struct** (Debug, Clone):
       - command_id: i64
       - session_id: crate::event::SessionId
       - summary: String
       - severity: String (Error/Warning/Info/Success)
       - timestamp_secs: u64 (SystemTime::now().duration_since(UNIX_EPOCH))
       - token_cost: usize
       - collapsed_count: u32

    2. **ActivityStreamConfig struct** (Debug, Clone, Default):
       - channel_capacity: usize (default 256)
       - token_budget: usize (default 4096)
       - max_rate_per_sec: f64 (default 5.0)

    3. **estimate_tokens(summary: &str) -> usize**: `(summary.len() / 4) + 8`

    4. **ActivityFilter struct**:
       - config: ActivityStreamConfig
       - window: VecDeque<ActivityEvent>
       - window_token_cost: usize
       - rate_tokens: f64 (init to max_rate_per_sec)
       - last_refill: Instant
       - last_fingerprint: Option<String>
       - pending_collapsed: u32

    5. **ActivityFilter::new(config)** — initialize with rate_tokens = config.max_rate_per_sec (full bucket)

    6. **ActivityFilter::process(&mut self, command_id, session_id, summary, severity) -> Option<ActivityEvent>**:
       a. Refill rate tokens: elapsed * max_rate_per_sec, capped at max_rate_per_sec
       b. Check rate limit: if rate_tokens < 1.0, return None (drop)
       c. Compute fingerprint = format!("{}:{}", severity, summary)
       d. If severity is "Success" or "Info" AND fingerprint == last_fingerprint: increment pending_collapsed, return None
       e. Otherwise: if pending_collapsed > 0, update the last event in window's collapsed_count to (1 + pending_collapsed), then reset pending_collapsed = 0
       f. Consume rate token (subtract 1.0, floor at 0.0)
       g. Build ActivityEvent with token_cost = estimate_tokens(&summary), collapsed_count = 1, timestamp_secs from SystemTime
       h. Push to budget window: add token_cost to window_token_cost, push_back. While window_token_cost > token_budget, pop_front and subtract.
       i. Store fingerprint as last_fingerprint
       j. Return Some(event)

    7. **ActivityFilter::flush_collapsed(&mut self) -> Option<ActivityEvent>**: If pending_collapsed > 0 and last event exists in window, update its collapsed_count and return a clone. Used when the agent runtime drains. (Optional helper for Phase 56.)

    8. **ActivityFilter::window_events(&self) -> impl Iterator<Item = &ActivityEvent>**: self.window.iter()

    9. **create_channel(config: &ActivityStreamConfig) -> (std::sync::mpsc::SyncSender<ActivityEvent>, std::sync::mpsc::Receiver<ActivityEvent>)**: Calls sync_channel(config.channel_capacity).

    10. Add `pub mod activity_stream;` to lib.rs.

    IMPORTANT: Use `try_send()` not `send()` for the SyncSender -- blocking the main thread is prohibited. Document this in the create_channel doc comment.

    Rate limiter note: The bucket starts full (max_rate_per_sec tokens). Each process() call refills based on elapsed time, then checks. This means the first N calls in a burst pass through (where N = max_rate_per_sec), then subsequent calls within the same second are dropped.

    Collapse strategy: Only Success/Info severity events are candidates for collapse. Error/Warning events always pass through (they are actionable). When a non-matching event arrives and pending_collapsed > 0, the last window event's collapsed_count is updated retroactively.
