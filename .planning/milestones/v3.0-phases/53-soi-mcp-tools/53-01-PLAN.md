    1. **Add `get_last_n_run_ids` to `HistoryDb` in `crates/glass_history/src/db.rs`:**
       - Add method `pub fn get_last_n_run_ids(&self, command_pattern: &str, n: usize) -> Result<Vec<i64>>`
       - SQL: `SELECT id FROM commands WHERE command LIKE ?1 ORDER BY started_at DESC LIMIT ?2`
       - Reverse result vec after fetch so it is oldest-first (forward progression for diffs)
       - Add test in existing `#[cfg(test)] mod tests` verifying ordering and LIKE pattern matching

    2. **Add parameter structs in `crates/glass_mcp/src/tools.rs`:**
       - `QueryParams`: command_id (i64), budget (Option<String>), severity (Option<String>), file (Option<String>), record_type (Option<String>)
       - `QueryTrendParams`: command (String, with schemars description noting LIKE `%` wildcards), n (Option<i64>, default 5)
       - `QueryDrillParams`: record_id (i64)
       - All structs: `#[derive(Debug, Deserialize, schemars::JsonSchema)]` with `#[schemars(description = "...")]` on each field

    3. **Add `parse_budget` helper function** (module-level, not a method):
       ```rust
       fn parse_budget(s: Option<&str>) -> glass_history::compress::TokenBudget {
           match s.unwrap_or("summary") {
               "one_line" => TokenBudget::OneLine,
               "detailed" => TokenBudget::Detailed,
               "full" => TokenBudget::Full,
               _ => TokenBudget::Summary,
           }
       }
       ```

    4. **Implement `glass_query` tool method on GlassServer:**
       - Description (< 40 words): "Query structured output records for a command at a token budget level. Returns compressed summary with record_ids for drill-down. Budget: one_line, summary, detailed, full."
       - spawn_blocking: open HistoryDb, call `db.compress_output(command_id, budget)`
       - If None: return `Content::text("No SOI data for command {id}. Command may predate SOI integration or had no structured output.")`
       - If Some: return `Content::json(&compressed_output)`

    5. **Implement `glass_query_trend` tool method on GlassServer:**
       - Description (< 40 words): "Compare last N runs of a command pattern to detect regressions. Use % wildcards for pattern matching. Returns per-run summaries and diffs showing new/resolved issues."
       - spawn_blocking: open HistoryDb
       - Call `db.get_last_n_run_ids(&params.command, n)` where n defaults to 5
       - If < 2 runs: return `Content::text("Need at least 2 runs to compare. Found {count} run(s) for '{command}'.")`
       - For each run: call `db.compress_output(id, TokenBudget::Summary)` to get per-run summary
       - For each consecutive pair: fetch records with `db.get_output_records(id, None, None, None, 10000)` then call `glass_history::compress::diff_compress(curr, Some(&prev))`
       - Build JSON response: `{ runs: [{command_id, summary}], diffs: [{from_id, to_id, new_count, resolved_count, change_line, new_records, resolved_records}], regression_detected: bool }`
       - Regression detection: any diff has `new_count > 0` AND any `new_records` entry has `record_type == "TestResult"` AND `severity == "Error"`
       - Return `Content::json(&result)`

    6. **Implement `glass_query_drill` tool method on GlassServer:**
       - Description (< 40 words): "Expand a specific output record to full detail. Pass a record_id from glass_query's record_ids field. Returns complete record data including context lines and failure details."
       - spawn_blocking: open HistoryDb, inline SQL query:
         `SELECT id, command_id, record_type, severity, file_path, data FROM output_records WHERE id = ?1`
       - Use `.optional()` from rusqlite
       - If None: return `Content::text("No record found with id {record_id}. Use record_ids from glass_query response.")`
       - If Some: deserialize `data` column as `serde_json::Value`, build response JSON with all fields, return `Content::json(&response)`

    7. **Keep tool descriptions concise** -- the STATE.md concern about tool token footprint means each description should be under 40 words. Do NOT add verbose examples in tool descriptions.
