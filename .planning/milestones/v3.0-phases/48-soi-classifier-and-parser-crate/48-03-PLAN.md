pub struct ParsedOutput {
    pub output_type: OutputType,
    pub summary: OutputSummary,
    pub records: Vec<OutputRecord>,

    pub raw_byte_count: usize,

    PackageEvent { action: String, package: String, version: Option<String>, detail: Option<String> },
    FreeformChunk { text: String, line_count: usize },
}
pub enum Severity { Error, Warning, Info, Success }
pub enum TestStatus { Passed, Failed, Skipped, Ignored }

1. `pub fn parse(output: &str) -> ParsedOutput`:

     - NPM_ADDED: `(?i)added (\d+) packages?`
     - NPM_REMOVED: `(?i)removed (\d+) packages?`
     - NPM_AUDITED: `(?i)audited (\d+) packages?`
     - NPM_VULNS: `(\d+) vulnerabilit(?:y|ies)`
     - NPM_VULN_DETAIL: `(\d+) (critical|high|moderate|low)`
     - NPM_DEPRECATED: `npm warn deprecated ([^:]+):\s*(.+)`
   - Process output line by line. For each regex match, create a PackageEvent record:
     - "added N" -> PackageEvent { action: "added", package: format!("{} packages", n), version: None, detail: None }
     - "removed N" -> PackageEvent { action: "removed", ... }
     - "audited N" -> PackageEvent { action: "audited", ... }
     - Vulnerability line -> PackageEvent { action: "vulnerabilities", package: String::new(), version: None, detail: Some(breakdown) }. Parse the detail breakdown (e.g., "1 moderate, 2 high") from NPM_VULN_DETAIL captures.
     - Deprecated warnings -> PackageEvent { action: "deprecated", package: pkg_name, version: None, detail: Some(msg) }
   - Build OutputSummary from collected events. Severity: if any critical/high vulns -> Error; if moderate/low -> Warning; else Success.
   - If no records matched at all, return freeform fallback.

**Part A - pytest parser** (`crates/glass_soi/src/pytest.rs`):

1. `pub fn parse(output: &str) -> ParsedOutput`:
   - Use OnceLock<Regex> patterns (from RESEARCH.md):
     - PYTEST_RESULT: `^(.+::[\w\[\]]+)\s+(PASSED|FAILED|ERROR|SKIPPED|XFAIL|XPASS)`
     - PYTEST_SUMMARY: `(\d+) passed(?:, (\d+) failed)?(?:, (\d+) error)?.*in ([\d.]+)s`
     - PYTEST_FAILURE_HEADER: `^_{3,}\s+(.+)\s+_{3,}$` (pytest failure section headers)
     - PYTEST_SHORT_SUMMARY: `^FAILED (.+) - (.+)$`
   - Parse line by line:
     - PYTEST_RESULT match: create TestResult with name, map PASSED->Passed, FAILED->Failed, ERROR->Failed, SKIPPED->Skipped, XFAIL->Skipped, XPASS->Passed
     - Track failure sections for failure_message extraction (between FAILURES header and summary line)
     - PYTEST_SHORT_SUMMARY: attach as failure_message to matching test name
     - PYTEST_SUMMARY: create TestSummary record (map "error" count into failed, duration in seconds -> ms)
   - Build OutputSummary. Severity: Error if any failed, Success otherwise.

   - Standard pytest output with mixed results (from RESEARCH.md fixture)
   - All passing
   - pytest with SKIPPED and XFAIL markers

**Part B - jest parser** (`crates/glass_soi/src/jest.rs`):

1. `pub fn parse(output: &str) -> ParsedOutput`:
   - First: `let clean = crate::ansi::strip_ansi(output);` -- jest always needs ANSI stripping
   - Use OnceLock<Regex> patterns:
     - JEST_SUITE: `^(PASS|FAIL)\s+(.+)$` (suite-level results)
     - JEST_TEST_PASS: `^\s+[✓✔]\s+(.+?)(?:\s+\((\d+)\s*m?s\))?$` (individual passing test with optional duration)
     - JEST_TEST_FAIL: `^\s+[✕✗×]\s+(.+?)(?:\s+\((\d+)\s*m?s\))?$` (individual failing test)
     - JEST_SUMMARY: `Tests:\s+(?:(\d+) failed,\s+)?(\d+) passed,\s+(\d+) total`
     - JEST_TIME: `Time:\s+([\d.]+)\s*s`
   - Parse line by line on the ANSI-stripped text:
     - JEST_SUITE: track current suite name (for prefixing test names)
     - JEST_TEST_PASS: TestResult with name = "{suite} > {test_name}", Passed, duration_ms from capture
     - JEST_TEST_FAIL: TestResult with name, Failed, duration_ms
     - Failure blocks: after a failing test, collect indented lines until next test line as failure_message. Jest shows expected/received diffs here.
     - JEST_SUMMARY: TestSummary record
     - JEST_TIME: total_duration_ms on TestSummary
   - Build OutputSummary. Severity: Error if any failed, Success if all passed.
