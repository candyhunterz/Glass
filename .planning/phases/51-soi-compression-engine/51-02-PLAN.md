<objective>
Add diff-aware compression that compares the current run's output records against the most recent prior run of the same command, producing a change summary with new/resolved issue counts.

Purpose: AI agents and MCP tools can see "compared to last run: 2 new errors, 1 resolved" rather than a static snapshot, enabling faster triage of regressions.
Output: `diff_compress()` function, `DiffSummary`/`RecordFingerprint` types, and `get_previous_run_records()` DB helper.
</objective>

**In `crates/glass_history/src/soi.rs`**, add a new public function:

```rust
/// Fetch output records for the most recent prior run of the same command text.
///
/// Excludes `current_command_id` so the current run is never compared to itself.
/// Returns `Ok(None)` if no prior run exists (first time this command ran).
pub fn get_previous_run_records(
    conn: &Connection,
    command_text: &str,
    current_command_id: i64,
) -> Result<Option<Vec<OutputRecordRow>>>
```

Implementation: Query `SELECT id FROM commands WHERE command = ?1 AND id != ?2 ORDER BY started_at DESC LIMIT 1` to find the previous command id. Use `OptionalExtension` (already imported in db.rs but needs import in soi.rs: `use rusqlite::OptionalExtension;`). If found, call `get_output_records(conn, prev_id, None, None, None, 10000)`. Return `Ok(Some(records))`. If not found, return `Ok(None)`.

**In `crates/glass_history/src/db.rs`**, add delegation method after `compress_output`:

```rust
/// Fetch output records for the most recent prior run of the same command.
pub fn get_previous_run_records(
    &self,
    command_text: &str,
    current_command_id: i64,
) -> Result<Option<Vec<crate::soi::OutputRecordRow>>> {
    crate::soi::get_previous_run_records(&self.conn, command_text, current_command_id)
}
```

**In `crates/glass_history/src/compress.rs`**, add:

1. **RecordFingerprint struct** with fields: record_type (String), severity (Option<String>), file_path (Option<String>), message_prefix (String). Derive Debug, Clone, PartialEq, Eq, Hash, Serialize.

2. **DiffSummary struct** with fields: new_records (Vec<RecordFingerprint>), resolved_records (Vec<RecordFingerprint>), new_count (usize), resolved_count (usize), change_line (String). Derive Debug, Clone, Serialize.

3. **diff_compress() function**:
```rust
pub fn diff_compress(
    current_records: &[crate::soi::OutputRecordRow],
    previous_records: Option<&[crate::soi::OutputRecordRow]>,
) -> DiffSummary
```

Logic:
- If `previous_records` is `None`: return DiffSummary with empty vecs and `change_line = "first run -- no comparison available"`.
- If `previous_records` is `Some(prev)` and `prev.is_empty()`: return `change_line = "no structured data for previous run"`.
- Otherwise: Build `HashSet<RecordFingerprint>` for current and previous. New = current - previous. Resolved = previous - current. Format change_line as "compared to last run: {N} new, {M} resolved". Skip FreeformChunk records in fingerprint computation (no stable identity).

4. **fingerprint() helper function**: `fn fingerprint(row: &OutputRecordRow) -> Option<RecordFingerprint>`. Returns None for FreeformChunk. Parses `row.data` as `serde_json::Value` to extract message_prefix (first 80 chars of the identity field: "message" for CompilerError/GenericDiagnostic, "name" for TestResult, "package" for PackageEvent). If JSON parse fails, use empty string for message_prefix.

5. Add re-exports in `crates/glass_history/src/lib.rs`: `pub use compress::{DiffSummary, RecordFingerprint, diff_compress};` (append to existing compress re-exports).

6. Write all test functions from behavior block. The DB-dependent tests (get_previous_run_records_*) use the same `make_db()` + `sample_command()` pattern from soi.rs tests. The pure diff tests construct OutputRecordRow values directly.
