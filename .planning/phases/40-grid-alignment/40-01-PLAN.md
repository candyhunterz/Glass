Rewrite `GridRenderer::new()` to derive cell_height from font metrics instead of the 1.2x multiplier:

1. Create initial metrics with `Metrics::new(physical_font_size, physical_font_size)` (NOT `physical_font_size * 1.2`)
2. Shape "M" with a measure buffer (existing pattern)
3. Extract `cell_width` from `run.glyphs.first().map(|g| g.w)` (existing)
4. Extract `cell_height` from `run.line_height.max(physical_font_size).ceil()` (NEW -- replaces hardcoded 1.2x)
5. Store cell_height from font metrics

Then rewrite `build_text_buffers()` into `build_cell_buffers()` -- one Buffer per non-empty cell:

1. For each cell in snapshot.cells:
   - Skip if `flags.contains(Flags::WIDE_CHAR_SPACER)`
   - Skip if `cell.c == ' ' && cell.zerowidth.is_empty()`
   - Create `Buffer::new(font_system, metrics)` with metrics using the new cell_height
   - Call `buffer.set_size(font_system, Some(self.cell_width), Some(self.cell_height))`
   - Call `buffer.set_monospace_width(font_system, Some(self.cell_width))` for grid snapping
   - Use `cell.c.encode_utf8(&mut char_buf)` for zero-alloc single-char encoding (stack `[u8; 4]`)
   - Only allocate String when `cell.zerowidth` is non-empty (push char + zero-width chars)
   - Set attrs with family, color (fg), bold (Weight::BOLD), italic (Style::Italic)
   - Call `buffer.shape_until_scroll(font_system, false)`
   - Push buffer to vec

2. Also push a `CellPosition { column: usize, line: i32 }` alongside each buffer to track grid position (avoids the buffer-TextArea index mismatch pitfall from research). Store these in a parallel Vec or return them alongside. A simple approach: add a `cell_positions: Vec<(usize, i32)>` parameter alongside the buffers vec.

Then rewrite `build_text_areas_offset()` to use cell positions instead of line_idx enumeration:

1. Accept `cell_positions: &[(usize, i32)]` parameter alongside buffers
2. For each (buffer, (col, line)) pair:
   - `left = x_offset + col as f32 * self.cell_width`
   - `top = y_offset + line as f32 * self.cell_height`
   - Create TextArea with `scale: 1.0` (never use TextArea.scale for DPI per glyphon issue #117)

Update `build_text_areas()` to call the new offset version with (0.0, 0.0).

Keep `build_rects()`, `build_rects_offset()`, `build_selection_rects()` unchanged -- they already use cell_width/cell_height correctly and will automatically benefit from the new cell_height value.

Remove the old per-line `build_text_buffers()` method entirely.

Add `#[cfg_attr(feature = "perf", tracing::instrument(skip_all))]` to `build_cell_buffers` for consistency.
