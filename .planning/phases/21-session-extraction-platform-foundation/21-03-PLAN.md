  <files>src/main.rs, src/search_overlay.rs</files>
  <action>
    This is the most critical task in Phase 21. It must be done MECHANICALLY to avoid regression. Do NOT rewrite event handling logic -- only change how fields are accessed.

    **Step 1: Update WindowContext struct definition**

    Replace the 15 session fields with one `session_mux: SessionMux`:
    ```rust
    struct WindowContext {
        window: Arc<Window>,
        renderer: GlassRenderer,
        frame_renderer: FrameRenderer,
        session_mux: SessionMux,
        first_frame_logged: bool,
    }
    ```

    Add imports: `use glass_mux::{SessionMux, Session, SessionId};`
    Remove: `use crate::search_overlay::SearchOverlay;` -- now comes from glass_mux.

    **Step 2: Update resumed() to create Session and SessionMux**

    In the `resumed()` handler where WindowContext is constructed:
    1. Generate a SessionId: `let session_id = SessionId::new(0);` (first session always 0)
    2. Pass session_id to EventProxy::new() (which Plan 02 updated to accept it)
    3. Construct Session with all the fields that were previously inline in WindowContext
    4. Create SessionMux: `let session_mux = SessionMux::new(session);`
    5. Construct WindowContext with session_mux instead of inline fields

    **Step 3: Create a helper method for accessing the focused session**

    Add to WindowContext impl:
    ```rust
    fn session(&self) -> &Session {
        self.session_mux.focused_session().expect("no focused session")
    }
    fn session_mut(&mut self) -> &mut Session {
        self.session_mux.focused_session_mut().expect("no focused session")
    }
    ```

    **Step 4: Mechanically replace all field accesses**

    This is the bulk of the work. The event handler in `user_event()` (~400 lines) and the render path access WindowContext fields ~50+ times. For each occurrence:

    - `ctx.pty_sender` -> `ctx.session().pty_sender` (or `ctx.session_mut().pty_sender`)
    - `ctx.term` -> `ctx.session().term`
    - `ctx.default_colors` -> `ctx.session().default_colors`
    - `ctx.block_manager` -> `ctx.session_mut().block_manager`
    - `ctx.status` -> `ctx.session_mut().status`
    - `ctx.history_db` -> `ctx.session_mut().history_db`
    - `ctx.last_command_id` -> `ctx.session_mut().last_command_id`
    - `ctx.command_started_wall` -> `ctx.session_mut().command_started_wall`
    - `ctx.search_overlay` -> `ctx.session_mut().search_overlay`
    - `ctx.snapshot_store` -> `ctx.session_mut().snapshot_store`
    - `ctx.pending_command_text` -> `ctx.session_mut().pending_command_text`
    - `ctx.active_watcher` -> `ctx.session_mut().active_watcher`
    - `ctx.pending_snapshot_id` -> `ctx.session_mut().pending_snapshot_id`
    - `ctx.pending_parse_confidence` -> `ctx.session_mut().pending_parse_confidence`
    - `ctx.cursor_position` -> `ctx.session_mut().cursor_position`

    CRITICAL BORROW CHECKER ISSUE: When the code accesses both session fields AND window/renderer (e.g., `ctx.session_mut().block_manager` and `ctx.window.request_redraw()`), you cannot hold a mutable reference to session while borrowing ctx immutably for window. Solution: Extract session data into local variables FIRST, then use window/renderer. For example:
    ```rust
    // BEFORE (won't compile):
    ctx.session_mut().block_manager.do_thing();
    ctx.window.request_redraw();

    // AFTER (compiles):
    {
        let session = ctx.session_mux.focused_session_mut().unwrap();
        session.block_manager.do_thing();
    }
    ctx.window.request_redraw();
    ```

    Or use field-level splitting:
    ```rust
    let SessionMux { .. } = &mut ctx.session_mux;
    // Rust can see session_mux and window are disjoint
    ```

    **Step 5: Update AppEvent match arms for session routing**

    Plan 02 added session_id to AppEvent variants. Now use it for routing:
    ```rust
    AppEvent::Shell { window_id, session_id, event: shell_event, line } => {
        if let Some(ctx) = self.windows.get_mut(&window_id) {
            // In single-session mode, we could ignore session_id and use focused_session_mut().
            // But for correctness, route by session_id:
            if let Some(session) = ctx.session_mux.session_mut(session_id) {
                // ... existing shell event handling, replacing ctx.field with session.field
            }
        }
    }
    ```

    For TerminalDirty (no session_id), keep as-is -- just request redraw.

    **Step 6: Update find_shell_integration() to be platform-aware**

    The existing `find_shell_integration()` function (near end of main.rs) is Windows-only (looks for glass.ps1). Make it platform-aware using glass_mux::platform::default_shell():

    ```rust
    fn find_shell_integration(shell_name: &str) -> Option<std::path::PathBuf> {
        let script_name = if shell_name.contains("pwsh") || shell_name.to_lowercase().contains("powershell") {
            "glass.ps1"
        } else if shell_name.contains("zsh") {
            "glass.zsh"
        } else if shell_name.contains("fish") {
            "glass.fish"
        } else {
            "glass.bash"
        };
        // ... rest of path resolution (same as existing)
    }
    ```
