    **Step 2: Replace alacritty_terminal EventLoop with custom read loop** in pty.rs:

    Remove the use of `PtyEventLoop`. Instead, create a custom `glass_pty_loop` that:

    1. Takes ownership of: pty (from tty::new), term (Arc<FairMutex<Term>>), event_proxy (EventProxy), osc_scanner (OscScanner)
    2. Creates a `vte::Parser` for feeding bytes to the terminal
    3. Creates an `mpsc::channel` for receiving input/resize messages (replaces EventLoopSender)
    4. Runs on a dedicated `std::thread::spawn`:

    ```rust
    fn glass_pty_loop(
        mut pty: Pty,
        term: Arc<FairMutex<Term<EventProxy>>>,
        proxy: EventLoopProxy<AppEvent>,
        window_id: WindowId,
        rx: mpsc::Receiver<PtyMsg>,
    ) {
        let mut scanner = OscScanner::new();
        let mut parser = vte::Parser::new();
        let mut buf = [0u8; 4096];

        loop {
            // 1. Drain write queue (non-blocking)
            while let Ok(msg) = rx.try_recv() {
                match msg {
                    PtyMsg::Input(data) => { pty.writer().write_all(&data).ok(); }
                    PtyMsg::Resize(size) => { pty.on_resize(size); }
                    PtyMsg::Shutdown => return,
                }
            }

            // 2. Read from PTY (blocking, with small timeout via set_read_timeout or poll)
            match pty.reader().read(&mut buf) {
                Ok(0) => { /* EOF */ send TerminalExit; break; }
                Ok(n) => {
                    let data = &buf[..n];

                    // 3. Pre-scan for OSC sequences
                    let events = scanner.scan(data);
                    for osc_event in events {
                        let line = term.lock().cursor().point.line.0 as usize;
                        let shell_event = convert_osc_to_shell(osc_event);
                        proxy.send_event(AppEvent::Shell { window_id, event: shell_event, line }).ok();
                    }

                    // 4. Feed to alacritty_terminal parser
                    let mut term_guard = term.lock();
                    for byte in data {
                        parser.advance(&mut *term_guard, *byte);
                    }
                    drop(term_guard);

                    // 5. Notify main thread
                    proxy.send_event(AppEvent::TerminalDirty { window_id }).ok();
                }
                Err(e) if e.kind() == std::io::ErrorKind::WouldBlock => continue,
                Err(_) => { send TerminalExit; break; }
            }
        }
    }
    ```

    Note on `vte::Parser::advance`: the VTE parser's `advance` method takes a `&mut impl Perform` and a single byte. The `Term` type implements `vte::Perform` (via alacritty_terminal's handler). Verify the exact API -- it may be `parser.advance(&mut performer, byte)` where performer wraps term.

    Actually, alacritty_terminal uses `state.parser.advance(&mut **terminal, &buf[..n])` where it passes a byte slice. Check if vte::Parser has a method that takes a slice. If vte 0.15 uses `advance` per-byte, loop over bytes. If it has a slice method, use that.

    Update `spawn_pty` signature to accept `window_id: WindowId` (needed for AppEvent) and `proxy: EventLoopProxy<AppEvent>` (needed to send shell events). Return type changes from `EventLoopSender` to `mpsc::Sender<PtyMsg>` (custom channel instead of alacritty_terminal's).

    Define a local `PtyMsg` enum if needed (Input, Resize, Shutdown) or reuse alacritty_terminal's if compatible.

    Handle `drain_on_exit`: after the read loop exits, do a final drain of remaining PTY bytes.
