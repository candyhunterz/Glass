<task type="auto">
  <name>Task 1: Create BlockRenderer and StatusBarRenderer</name>
  <files>crates/glass_renderer/src/block_renderer.rs, crates/glass_renderer/src/status_bar.rs, crates/glass_renderer/src/lib.rs</files>
  <action>
    Create block_renderer.rs:

    1. `BlockRenderer` struct (stateless helper, takes cell dimensions as constructor args):
       - `cell_width: f32, cell_height: f32`

    2. `build_block_rects(&self, blocks: &[&Block], display_offset: usize, screen_lines: usize, viewport_width: f32) -> Vec<RectInstance>`:
       - For each visible block, generate:
         a. A horizontal separator line (1px tall, full width) at the block's prompt_start_line position, in a subtle gray (RGB 60,60,60, alpha 1.0)
         b. If block has exit_code: a small badge rect at the right edge of the separator line:
            - exit_code == 0: green background (RGB 40,160,40)
            - exit_code != 0: red background (RGB 200,50,50)
            - Badge size: ~3 cells wide, 1 cell tall, right-aligned
       - Convert block line numbers to pixel positions: `y = (line - display_offset) * cell_height`
       - Skip blocks whose separator would be off-screen

    3. `build_block_text(&self, blocks: &[&Block], display_offset: usize, cell_height: f32) -> Vec<BlockLabel>`:
       - For each complete block with exit_code, produce a `BlockLabel` with:
         - Position (x, y) for the badge text (checkmark/X character)
         - The text string: "checkmark" for 0, "X" for non-zero
         - Color: white
       - For each complete block with duration, produce a `BlockLabel` with:
         - Position right-aligned, next to badge
         - Duration text from `format_duration()`
         - Color: subtle gray (RGB 140,140,140)
       - `BlockLabel` struct: x (f32), y (f32), text (String), color (Rgb)

    Create status_bar.rs:

    1. `StatusBarRenderer` struct:
       - `cell_height: f32`

    2. `build_status_rects(&self, viewport_width: f32, viewport_height: f32) -> Vec<RectInstance>`:
       - One full-width rect at the bottom of viewport, 1 cell_height tall
       - Background color: slightly lighter than terminal bg (RGB 38,38,38)

    3. `build_status_text(&self, cwd: &str, git_info: Option<&GitInfo>, viewport_height: f32) -> StatusLabel`:
       - Left side: CWD path (truncated to fit if needed)
       - Right side: git branch name + dirty count if available, e.g., "main +3"
       - `StatusLabel` struct: left_text (String), right_text (Option<String>), y (f32), left_color (Rgb), right_color (Rgb)
       - CWD color: white (RGB 204,204,204)
       - Git branch color: cyan (RGB 80,200,200), dirty count in yellow (RGB 200,200,80) if dirty_count > 0

    Update lib.rs: add `pub mod block_renderer;` and `pub mod status_bar;` with re-exports.
  </action>
  <verify>
    <automated>cargo build -p glass_renderer 2>&1</automated>
  </verify>
  <done>BlockRenderer produces separator rects and badge rects for visible blocks. StatusBarRenderer produces background rect and text labels. Both compile cleanly against Plan 01 types.</done>

<task type="auto">
  <name>Task 2: Integrate block and status rendering into FrameRenderer</name>
  <files>crates/glass_renderer/src/frame.rs, crates/glass_renderer/src/grid_renderer.rs</files>
  <action>
    Modify FrameRenderer to accept block and status data during draw_frame:

    1. Add fields to FrameRenderer:
       - `block_renderer: BlockRenderer`
       - `status_bar: StatusBarRenderer`
       Initialize both in FrameRenderer::new() using cell dimensions from GridRenderer.

    2. Extend `draw_frame` signature to accept optional block/status data:
       ```rust
       pub fn draw_frame(
           &mut self,
           device: &wgpu::Device,
           queue: &wgpu::Queue,
           view: &wgpu::TextureView,
           width: u32,
           height: u32,
           snapshot: &GridSnapshot,
           blocks: &[&Block],       // empty slice if no blocks yet
           status: Option<&StatusState>,
       )
       ```

    3. In the draw pipeline, AFTER building grid rects but BEFORE submitting:
       a. Call `block_renderer.build_block_rects()` to get separator and badge rects
       b. Call `status_bar.build_status_rects()` to get status background rect
       c. Append all these rects to the existing rect_instances vec
       d. For block text labels (badge text, duration text): create additional glyphon Buffer entries and TextAreas
       e. For status bar text: create a glyphon Buffer for CWD and git text, add as TextArea

    4. The text for badges, durations, and status bar should use the same font system (glyph_cache.font_system) but at a slightly smaller size or the same size -- use same size for simplicity, create additional line buffers.

    5. Status bar positioning: the status bar rect sits at `y = height - cell_height`. Terminal content area is effectively `height - cell_height` pixels tall. NOTE: Do NOT change PTY resize here -- that is handled in Plan 04 wiring. Just render the status bar overlapping the last line for now.

    **Important:** Keep backward compatibility. When blocks is empty and status is None, draw_frame behaves identically to Phase 2.
  </action>
