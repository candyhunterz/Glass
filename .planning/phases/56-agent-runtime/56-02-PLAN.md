<task type="auto">
  <name>Task 1: AgentRuntime struct and subprocess spawn in Processor</name>
  <files>src/main.rs</files>
  <action>
    Create an `AgentRuntime` struct (defined inline in `src/main.rs` or as a private module) that encapsulates the agent subprocess lifecycle. This struct lives as a field on Processor.

    **AgentRuntime struct fields:**
    - `child: Option<std::process::Child>` — the claude subprocess
    - `cooldown: CooldownTracker` — from glass_core::agent_runtime
    - `budget: BudgetTracker` — from glass_core::agent_runtime
    - `config: AgentRuntimeConfig` — from glass_core::agent_runtime
    - `restart_count: u32` — for exponential backoff (max 3 restarts)
    - `last_crash: Option<std::time::Instant>` — for backoff timing

    **Processor new fields:**
    - `agent_runtime: Option<AgentRuntime>` — None when agent.mode == Off
    - `agent_cost_usd: f64` — cumulative cost this session
    - `agent_proposals_paused: bool` — true when budget exceeded
    - Remove `#[allow(dead_code)]` from `activity_stream_rx` since Plan 02 now consumes it.

    **Spawn logic (called from `watcher_spawned` init block, after activity stream channel creation):**
    1. Check if `claude` binary exists on PATH using `which::which("claude")` or `std::process::Command::new("claude").arg("--version").output()`. If not found, log warning and skip agent init (AGTR-04 graceful degradation).
    2. Write system prompt to `~/.glass/agent-system-prompt.txt`. The prompt instructs the agent: "You are Glass Agent. You monitor terminal activity and propose fixes. Output proposals as GLASS_PROPOSAL: {json}. Available tools: glass_query, glass_context." Keep it concise (~200 words).
    3. Write MCP config to `~/.glass/agent-mcp.json` pointing to Glass's MCP server. For Phase 56, use a minimal config with the Glass MCP stdio transport. If the MCP server path is not available at spawn time, skip MCP config and omit `--mcp-config` flag.
    4. Build command args via `build_agent_command_args()`.
    5. Create `Command::new("claude")` with args, `Stdio::piped()` for stdin/stdout, `Stdio::null()` for stderr.
    6. Apply platform orphan prevention BEFORE spawn (see Task 2 for details).
    7. Spawn the child process.
    8. Take `child.stdout` and spawn a reader thread named "glass-agent-reader" that:
       - Uses `BufReader::new(stdout).lines()` to read JSON lines
       - For each line, parse with serde_json::Value
       - On `type=="result"`: extract cost_usd via `parse_cost_from_result`, send `AppEvent::AgentQueryResult { cost_usd }` via EventLoopProxy
       - On `type=="assistant"`: extract text content blocks, concatenate text, call `extract_proposal()`, if Some send `AppEvent::AgentProposal(proposal)` via EventLoopProxy
       - On EOF/error: send `AppEvent::AgentCrashed` via EventLoopProxy, break
    9. Take `child.stdin` and `activity_stream_rx.take()`, spawn a writer thread named "glass-agent-writer" that:
       - Wraps stdin in `BufWriter`
       - Iterates `rx.iter()` (blocking receive)
       - Checks `should_send_in_mode(config.mode, &event.severity)` — skip if false
       - Checks cooldown via shared state or inline Instant tracking — skip if within cooldown
       - Calls `format_activity_as_user_message(&event)` and `writeln!` + `flush()`
       - On write error (BrokenPipe): break

    **Event handlers in Processor::handle_event (or equivalent match arm):**
    - `AppEvent::AgentProposal(proposal)`: Log the proposal. Store in a `Vec<AgentProposalData>` field on Processor for Phase 58 UI. For now, just log at info level: "Agent proposal: {description}".
    - `AppEvent::AgentQueryResult { cost_usd }`: Add to `self.agent_cost_usd`. Check via BudgetTracker. If exceeded, set `agent_proposals_paused = true` and log warning.
    - `AppEvent::AgentCrashed`: Log error. If `restart_count < 3` and enough time since `last_crash` (exponential backoff: 5s, 15s, 45s), attempt re-spawn. Otherwise log "Agent restart limit reached, disabling agent."

    **Shutdown logic:** On app exit (in the Drop path or explicit shutdown), drop the stdin writer (causes EOF to child), call `child.try_wait()` then `child.kill()` if still running after 2 seconds.

    **Important:** The spawn MUST happen on the winit main thread (where Processor methods run), NOT inside a tokio::spawn or spawn_blocking closure. This is critical for the Unix prctl pattern (see research Pitfall 4).

  <name>Task 2: Platform orphan prevention and status bar cost display</name>
  <files>src/main.rs, crates/glass_renderer/src/status_bar.rs</files>
  <action>
    **Part A: Windows Job Object (AGTR-05)**

    Add a `#[cfg(target_os = "windows")]` function `setup_windows_job_object()` in `src/main.rs` (or a private `agent_platform` module). This function:
    1. Calls `CreateJobObjectW(null, null)` to create a new Job Object
    2. Sets `JOBOBJECT_EXTENDED_LIMIT_INFORMATION` with `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE`
    3. Assigns the current process to the Job Object via `AssignProcessToJobObject`
    4. Returns the HANDLE which must be stored in Processor for the app lifetime (dropping closes the handle which triggers kill-on-close)
    5. On failure, log warning and return None — do not crash

    Use `windows_sys::Win32::System::JobObjects::*` and `windows_sys::Win32::Foundation::*`. The workspace Cargo.toml features were already updated in Plan 01.

    Store the handle as `job_object_handle: Option<isize>` on Processor (isize is the HANDLE type in windows-sys). Call `setup_windows_job_object()` early in Processor initialization (before agent spawn).

    **Part B: Unix prctl (AGTR-05)**

    Add a `#[cfg(unix)]` block in the agent spawn path that uses `std::os::unix::process::CommandExt::pre_exec` to call `libc::prctl(libc::PR_SET_PDEATHSIG, libc::SIGKILL)` on the child. This requires adding `libc` as a dependency.

    In workspace `Cargo.toml`, add `libc = "0.2"` to `[workspace.dependencies]`. In `src/main.rs` (or the root crate Cargo.toml), add `libc` as a `[target.'cfg(unix)'.dependencies]` entry.

    If adding libc creates complications, an acceptable fallback is to skip prctl and rely on explicit kill in the shutdown path. Document this as a known limitation on Unix.

    **Part C: Status bar agent cost display (AGTR-07)**

    1. Add two fields to `StatusLabel` in `crates/glass_renderer/src/status_bar.rs`:
       ```rust
       pub agent_cost_text: Option<String>,
       pub agent_cost_color: Rgb,
       ```

    2. Add `agent_cost_text: Option<&str>` parameter to `build_status_text()`. Position the agent cost text between coordination_text and right_text (or after coordination_text if no right_text).

    3. Color: green `Rgb { r: 80, g: 220, b: 120 }` when active, red `Rgb { r: 255, g: 80, b: 80 }` when paused.

    4. Default `agent_cost_color` to green in the builder. Let the caller pass `agent_paused: bool` to toggle the color.

    5. In `src/main.rs`, at every call site of `build_status_text()`, add the new `agent_cost_text` parameter:
       - If `agent_runtime.is_some()` and `agent_cost_usd > 0.0`: pass `Some(&budget.cost_text())` or `Some(&budget.paused_text())` depending on `agent_proposals_paused`
       - Otherwise: pass `None`

    6. In the renderer's text layout (likely in `frame.rs` or wherever StatusLabel fields are rendered), render `agent_cost_text` at the appropriate position. Follow the same pattern used for `coordination_text` — positioned as a separate segment with its own color.

    **Verification:** Run `cargo test --workspace` to ensure no regressions. Run `cargo clippy --workspace -- -D warnings` for lint check.
