   - `pending_command_text: Option<String>` -- stores extracted text between CommandExecuted and CommandFinished

3. In WindowContext initialization (around line 270-286), open SnapshotStore:
   ```rust
   let snapshot_store = {
       let glass_dir = glass_snapshot::resolve_glass_dir(std::path::Path::new(&cwd_string));
       match glass_snapshot::SnapshotStore::open(&glass_dir) {
           Ok(store) => Some(store),
           Err(e) => {
               tracing::warn!("Failed to open snapshot store: {}", e);
               None
           }
       }
   };
   ```
   And add `snapshot_store` and `pending_command_text: None` to the WindowContext init.

4. In the CommandExecuted handler (around line 606), AFTER `ctx.command_started_wall = Some(...)`, add command text extraction. Move the existing extraction code (lines 630-661) here. The code extracts text from `block.command_start_line` to `block.output_start_line` via the terminal grid. Store result in `ctx.pending_command_text = Some(command_text)`.

   IMPORTANT: The extraction must happen AFTER `ctx.block_manager.handle_event(&osc_event, line)` has processed the CommandExecuted event, because that is when output_start_line gets set on the block (confirmed in research: block_manager.rs line 100-110).

5. In the CommandFinished handler (around line 630), REPLACE the grid extraction block with:
   `let command_text = ctx.pending_command_text.take().unwrap_or_default();`
   Delete the entire block from line 630-661 that reads from the grid.

6. Add `use glass_snapshot;` at the top of main.rs with other imports.

CRITICAL NOTES:
- The grid extraction logic itself does NOT change -- same code, just moved earlier in the lifecycle.
- pending_command_text.take() ensures the text is consumed exactly once and cleared.
- If CommandExecuted never fires (edge case), CommandFinished gets empty string via unwrap_or_default -- same as current behavior when blocks.last() returns None.
