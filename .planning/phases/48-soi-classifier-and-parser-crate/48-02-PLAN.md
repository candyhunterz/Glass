<task type="auto" tdd="true">
  <name>Task 2: Implement cargo test parser</name>
  <files>crates/glass_soi/src/cargo_test.rs</files>
  <behavior>
    - "test foo::bar ... ok" line -> TestResult with name="foo::bar", status=Passed
    - "test foo::bar ... FAILED" line -> TestResult with name="foo::bar", status=Failed
    - "test foo::bar ... ignored" line -> TestResult with name="foo::bar", status=Ignored
    - Failure block "---- foo::bar stdout ----" followed by message -> failure_message populated on Failed TestResult
    - "test result: FAILED. 1 failed; 3 passed; 1 ignored" -> TestSummary record
    - Duration "finished in 0.02s" -> total_duration_ms on TestSummary
    - Compilation failure (no "running N tests" line) -> delegates to cargo_build::parse
    - Empty output -> freeform fallback, no crash
    - Summary one_line: "N passed, M failed" (or "all N passed" if no failures)
  </behavior>
  <action>
Implement `crates/glass_soi/src/cargo_test.rs`:

1. `pub fn parse(output: &str) -> ParsedOutput`:
   - First check: does output contain "running \d+ tests?" line? If not, this is likely a compilation failure -- delegate to `super::cargo_build::parse(output, Some("cargo test"))` to extract compiler errors instead.
   - Use OnceLock<Regex> for all patterns (matching glass_errors pattern):
     - TEST_LINE: `^test (.+) \.\.\. (ok|FAILED|ignored)$`
     - FAILURE_HEADER: `^---- (.+) stdout ----$`
     - TEST_SUMMARY: `test result: (?:ok|FAILED)\. (\d+) failed; (\d+) passed; (\d+) ignored`
     - TEST_DURATION: `finished in (\d+\.?\d*)s`
   - Parse line by line:
     - Match TEST_LINE: create TestResult with name, map "ok"->Passed, "FAILED"->Failed, "ignored"->Ignored
     - Track failure blocks: when FAILURE_HEADER seen, collect subsequent lines until next separator ("----" or "test result:") as failure_message for the named test
     - Match TEST_SUMMARY: create TestSummary record with counts
     - Match TEST_DURATION: set total_duration_ms on TestSummary (convert seconds to ms)
   - After parsing, attach failure_message to matching Failed TestResults by name
   - Build OutputSummary: one_line like "3 passed, 1 failed, 1 ignored" or "all 5 passed". Severity: Error if any failed, Success if all passed, Info if only ignored.
   - token_estimate: ~5 + records.len() * 8

2. Add #[cfg(test)] tests with fixture strings from RESEARCH.md:
   - Mixed results (passed + failed + ignored): verify all TestResult records and TestSummary
   - All passing: verify Success severity
   - Compilation failure (no "running" line, has "error[E"): verify delegation to cargo_build parser returns CompilerError records
   - Failure message extraction: verify failure_message on Failed test contains the panic message
   - Empty output: verify no crash, returns freeform
