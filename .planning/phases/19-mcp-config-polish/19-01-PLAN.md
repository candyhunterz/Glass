<task type="auto" tdd="true">
  <name>Task 2: GlassContext pipeline statistics</name>
  <files>crates/glass_mcp/src/context.rs</files>
  <behavior>
    - ContextSummary includes pipeline_count (i64), avg_pipeline_stages (f64), pipeline_failure_rate (f64)
    - Empty database returns pipeline_count=0, avg_pipeline_stages=0.0, pipeline_failure_rate=0.0
    - Database with 2 pipeline commands (3 stages + 2 stages, one failed) returns pipeline_count=2, avg_pipeline_stages=2.5, pipeline_failure_rate=0.5
    - Time-filtered query (after parameter) only counts pipelines within the window
    - Division by zero avoided: zero pipeline commands produces 0.0 for avg and failure rate
  </behavior>
  <action>
    Extend ContextSummary struct with three new fields:
    - pipeline_count: i64 (number of distinct commands that have pipe_stages entries)
    - avg_pipeline_stages: f64 (average number of stages per pipeline command)
    - pipeline_failure_rate: f64 (fraction of pipeline commands with non-zero exit_code)

    In build_context_summary(), add two new SQL queries after the existing directory query (both using the same optional WHERE c.started_at >= ?1 time filter):

    Query 1 -- pipeline count and avg stages:
    ```sql
    SELECT COUNT(DISTINCT ps.command_id),
           CAST(COUNT(*) AS REAL) / NULLIF(COUNT(DISTINCT ps.command_id), 0)
    FROM pipe_stages ps
    JOIN commands c ON c.id = ps.command_id
    [WHERE c.started_at >= ?1]
    ```

    Query 2 -- pipeline failure count (separate to avoid overcounting):
    ```sql
    SELECT COUNT(DISTINCT ps.command_id)
    FROM pipe_stages ps
    JOIN commands c ON c.id = ps.command_id
    WHERE c.exit_code != 0 [AND c.started_at >= ?1]
    ```

    Compute failure_rate = failed_pipeline_count as f64 / pipeline_count as f64, defaulting to 0.0 when pipeline_count is 0.

    Add tests in the existing #[cfg(test)] mod tests block:
    - test_pipeline_stats_empty_db: asserts all three new fields are 0/0.0
    - test_pipeline_stats_with_data: insert commands + pipe stages, verify correct counts
    - test_pipeline_stats_with_time_filter: verify after parameter filters pipeline stats
    - test_pipeline_stats_division_by_zero: no pipeline commands in window, verify 0.0 (not NaN/Inf)

    For tests, use the existing test_db() helper and insert() function. To insert pipe stages, use db.insert_pipe_stages(command_id, &stages).
  </action>
