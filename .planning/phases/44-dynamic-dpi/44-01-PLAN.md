The handler must:

1. Cast scale_factor to f32: `let scale = scale_factor as f32;`
2. Log: `tracing::info!("DPI scale factor changed to {scale}");`
3. Rebuild font metrics: `ctx.frame_renderer.update_font(&self.config.font_family, self.config.font_size, scale);`
4. Get window physical size: `let size = ctx.window.inner_size();`
5. Guard zero size: `if size.width > 0 && size.height > 0 {`
6. Resize wgpu surface: `ctx.renderer.resize(size.width, size.height);`
7. Resize active tab panes -- copy the exact pattern from the Resized handler (L989-1016):
   - If multi-pane: call `resize_all_panes()`
   - If single-pane: compute num_cols/num_lines from cell_size, send PtyMsg::Resize, resize term, notify block_manager
8. Resize background tab sessions -- copy the exact pattern from the Resized handler (L1018-1047):
   - Compute full_size WindowSize
   - Iterate background tab session IDs (filter out active_tab_index)
   - For each: send PtyMsg::Resize, resize term, notify block_manager
9. Close the size guard brace
10. Request redraw: `ctx.window.request_redraw();`

CRITICAL constraints:
- Do NOT use TextArea.scale for DPI (glyphon issue #117) -- this is already correct in update_font path
- The Resized handler (L981-1051) is idempotent so if a Resized event follows ScaleFactorChanged, the double-processing is harmless
- Keep the tracing::info log (remove the tracing::warn about "not yet supported")
