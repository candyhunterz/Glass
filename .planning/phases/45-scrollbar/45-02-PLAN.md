**1. Add scrollbar state to WindowContext struct:**

Define `ScrollbarDragInfo` struct near WindowContext:
```rust
struct ScrollbarDragInfo {
    /// Which pane's scrollbar is being dragged
    pane_id: SessionId,
    /// Y offset within the thumb where drag started (for smooth dragging)
    thumb_grab_offset: f32,
    /// The scrollbar track top Y position
    track_y: f32,
    /// The scrollbar track height
    track_height: f32,
    /// The current thumb height (for drag math)
    thumb_height: f32,
    /// History size at drag start
    history_size: usize,
}
```

Initialize both fields to `None` / `None` in WindowContext creation.

**2. CursorMoved handler (line ~1606) — add BEFORE existing selection drag logic:**

At the top of the CursorMoved handler, before `if ctx.mouse_left_pressed`:

a. **Handle active scrollbar drag:**
If `ctx.scrollbar_dragging.is_some()`, compute target scroll position from mouse_y:
- Get drag info, compute `effective_y = mouse_y - drag.thumb_grab_offset`
- `scrollable_track = drag.track_height - drag.thumb_height`
- If scrollable_track <= 0, return early
- `ratio = ((effective_y - drag.track_y) / scrollable_track).clamp(0.0, 1.0)`
- `target_offset = ((1.0 - ratio) * drag.history_size as f32) as usize`
- Get current display_offset from term, compute delta = `target_offset as i32 - current_offset as i32` (note: positive delta = scroll up into history)
- BUT `Scroll::Delta` convention in alacritty_terminal: positive = scroll up. Actually need to check: the existing MouseWheel handler shows `scroll_display(Scroll::Delta(lines))` where positive delta from mouse wheel = scroll up. So we need `delta = target_offset as i32 - current_display_offset as i32`.
- Actually simpler: compute target_offset, get current display_offset, set delta. But Scroll::Delta is relative. Use: get term lock, read `grid().display_offset()`, compute delta as `(target_offset as i32) - (current as i32)`, then `scroll_display(Scroll::Delta(delta))`.
- Request redraw, return early (don't process selection drag)

b. **Update hover state:**
For single-pane: compute scrollbar_x = viewport_width - SCROLLBAR_WIDTH, check if mouse is in scrollbar region. For multi-pane: iterate pane layouts, check each pane's scrollbar.
- Compute which pane's scrollbar (if any) the mouse is over using `ctx.frame_renderer.scrollbar().hit_test(...)`
- If hover state changed (different pane or None vs Some), update `ctx.scrollbar_hovered_pane` and request redraw.
- Use SessionId tracking: for single-pane, the hovered pane is the focused session; for multi-pane, compute from pane layout bounds.

**3. MouseInput Pressed (Left button, line ~1645) — add scrollbar hit-test BEFORE tab bar check:**

After the tab bar click check at the top (which returns early for tab bar clicks), add scrollbar hit-test:

For single-pane mode: compute scrollbar_x, use `ctx.frame_renderer.scrollbar().hit_test(mouse_x, mouse_y, scrollbar_x, grid_y_offset, pane_height, display_offset, history_size, screen_lines)`.

For multi-pane mode: iterate pane layouts, check each pane's scrollbar hit_test.

If hit:
- `ctx.mouse_left_pressed = false;` (prevent text selection)
- If `ScrollbarHit::Thumb`: Start drag — record `ScrollbarDragInfo` with `thumb_grab_offset = mouse_y - thumb_top_y` (compute thumb top from `compute_thumb_geometry`), track_y, track_height, thumb_height, history_size.
- If `ScrollbarHit::TrackAbove`: `term.scroll_display(Scroll::PageUp)`, request redraw.
- If `ScrollbarHit::TrackBelow`: `term.scroll_display(Scroll::PageDown)`, request redraw.
- If clicked pane is not focused (multi-pane), also call `ctx.session_mux.set_focused_pane(pane_id)`.
- `return;` to prevent fall-through to text selection.

**4. MouseInput Released (Left button, line ~1782):**

If `ctx.scrollbar_dragging.is_some()`:
- Set `ctx.scrollbar_dragging = None;`
- Request redraw.
- Do NOT copy selection to clipboard (the release was from scrollbar, not selection).
- Return early before clipboard_copy.

**5. Wire hover/drag state to draw calls:**

In the RedrawRequested handler:
- Single-pane draw_frame call (~line 809): pass `ctx.scrollbar_hovered_pane.is_some()` and `ctx.scrollbar_dragging.is_some()` as the scrollbar_hovered/scrollbar_dragging bools.
- Multi-pane draw_multi_pane_frame call (~line 924): build a `Vec<(bool, bool)>` from pane_data, where for each pane `(hovered == scrollbar_hovered_pane.contains(sid), dragging == scrollbar_dragging.as_ref().map(|d| d.pane_id) == Some(sid))`. Pass as `&scrollbar_state`.

**Import requirements:**
- `use glass_renderer::{ScrollbarHit, SCROLLBAR_WIDTH};` (add to existing glass_renderer imports)
- `use alacritty_terminal::grid::Dimensions;` may already be imported for `.display_offset()`
