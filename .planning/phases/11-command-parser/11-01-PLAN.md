<objective>
Build the POSIX command parser that identifies file modification targets from shell commands.

Purpose: Enable pre-exec snapshots for known destructive commands (rm, mv, cp, sed -i, chmod, git checkout, truncate) by parsing command text and extracting file path arguments. This is the core of SNAP-03 -- the parser is a pure function (command_text, cwd) -> ParseResult with no state or DB access.

Output: command_parser.rs module with POSIX parsing, redirect detection, path resolution, ParseResult/Confidence types, and comprehensive unit tests.
</objective>

6. Implement in `command_parser.rs` (all in one file, target ~250-350 lines for POSIX):
   - `fn tokenize(command_text: &str) -> Vec<String>` using `shlex::split()` with whitespace-split fallback
   - `fn base_command(cmd: &str) -> &str` strips path prefix (e.g., `/usr/bin/rm` -> `rm`)
   - `fn resolve_path(path_str: &str, cwd: &Path) -> PathBuf` -- if absolute (including POSIX `/` on Windows), return as-is; otherwise `cwd.join(path)`
   - `fn contains_unparseable_syntax(text: &str) -> bool` -- detects `$(`, backtick, `${`, ` | `, ` && `, ` || `, `;`, `for `, `while `, `if ` prefixes
   - `fn extract_redirect_targets(command_text: &str, cwd: &Path) -> Vec<PathBuf>` -- scans for `>` and `>>` outside quotes, extracts the following filename token
   - `fn dispatch_command(cmd: &str, args: &[String], cwd: &Path) -> ParseResult` -- match on command name
   - Per-command extractors: `parse_rm`, `parse_mv`, `parse_cp`, `parse_sed`, `parse_chmod`, `parse_git`, `parse_truncate`
   - `parse_git` dispatches on subcommand: checkout (with -- handling), restore, clean, reset are destructive; status, log, diff, show, branch etc. are ReadOnly; unknown subcommands are Low
   - Read-only command list: ls, dir, cat, type, head, tail, less, more, grep, rg, find, which, where, echo, printf, pwd, whoami, date, env, set, wc, file, stat, df, du, ps, top, htop
   - Main `parse_command` function: trim, check empty, check unparseable syntax, extract redirects, tokenize, dispatch, merge redirect targets into result

Key constraints from research:
- Do NOT expand globs -- if args contain `*`, `?`, `[`, treat as Confidence::Low
- Do NOT resolve `$HOME` or any variables
- For `parse_mv`: ALL arguments (sources + dest) are targets
- For `parse_cp`: destination file is the target (may be overwritten)
- For `parse_sed`: only with `-i` flag; skip `-e` and `-f` flag-value pairs
- For `git checkout` without `--`: return Low confidence (ambiguous branch vs file)
- WSL path handling: treat paths starting with `/` as absolute even on Windows
