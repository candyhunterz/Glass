<task type="auto">
  <name>Task 1: Create LICENSE file and Windows MSI packaging via cargo-wix</name>
  <files>LICENSE, wix/main.wxs</files>
  <action>
1. Create an MIT LICENSE file at the repo root. Use year 2026, copyright holder "Glass Contributors".

2. Install cargo-wix locally: `cargo install cargo-wix`

3. Run `cargo wix init --force` from the repo root to generate `wix/main.wxs`. This generates a WiX template with a stable UpgradeCode GUID.

4. Edit `wix/main.wxs` to customize:
   - Product Name: "Glass Terminal"
   - Manufacturer: "Glass Contributors"
   - Ensure the binary source path points to `target\release\glass.exe`
   - Add a PATH environment variable component so glass.exe is on the system PATH. Add a Component element with a stable GUID (generate one, e.g., using PowerShell `[guid]::NewGuid()`) containing an Environment element:
     ```xml
     <Component Id='Path' Guid='{GENERATED-GUID}' KeyPath='yes'>
       <Environment Id='PATH' Name='PATH' Value='[APPLICATIONFOLDER]' Permanent='no' Part='last' Action='set' System='yes'/>
     </Component>
     ```
   - Add `<ComponentRef Id='Path'/>` to the Feature element alongside the binary ComponentRef
   - Ensure the install directory name is "Glass Terminal" under ProgramFiles64Folder

5. Verify by running `cargo wix --no-build --nocapture` (this will fail on non-Windows or if binary not built, but on Windows after `cargo build --release` it should produce an MSI in target/wix/).

CRITICAL: The UpgradeCode GUID generated by `cargo wix init` must NEVER be changed after this commit. Document it in a comment in main.wxs.
  </action>
  <verify>
    <automated>test -f LICENSE && test -f wix/main.wxs && grep -q "UpgradeCode" wix/main.wxs && grep -q "Environment" wix/main.wxs && grep -q "PATH" wix/main.wxs</automated>
  </verify>
  <done>LICENSE file exists with MIT license. wix/main.wxs exists with stable UpgradeCode, Glass Terminal product name, binary reference, and PATH environment variable component.</done>
</task>

<task type="auto">
  <name>Task 2: Create macOS DMG packaging and Linux deb metadata</name>
  <files>packaging/macos/Info.plist, packaging/macos/build-dmg.sh, packaging/linux/glass.desktop, Cargo.toml</files>
  <action>
1. Create `packaging/macos/Info.plist` with:
   - CFBundleName: "Glass"
   - CFBundleDisplayName: "Glass Terminal"
   - CFBundleIdentifier: "com.glass.terminal"
   - CFBundleVersion and CFBundleShortVersionString: "0.1.0" (will be parameterized later by CI)
   - CFBundleExecutable: "glass"
   - CFBundlePackageType: "APPL"
   - LSMinimumSystemVersion: "11.0"
   - NSHighResolutionCapable: true

2. Create `packaging/macos/build-dmg.sh`:
   ```bash
   #!/bin/bash
   set -euo pipefail

   APP_NAME="Glass"
   VERSION="${1:-0.1.0}"
   BUNDLE_DIR="target/macos/${APP_NAME}.app"

   rm -rf "${BUNDLE_DIR}"
   mkdir -p "${BUNDLE_DIR}/Contents/MacOS"
   mkdir -p "${BUNDLE_DIR}/Contents/Resources"

   cp target/release/glass "${BUNDLE_DIR}/Contents/MacOS/glass"

   # Generate Info.plist with correct version from argument
   sed "s/0\\.1\\.0/${VERSION}/g" packaging/macos/Info.plist > "${BUNDLE_DIR}/Contents/Info.plist"

   hdiutil create -volname "${APP_NAME}" \
       -srcfolder "${BUNDLE_DIR}" \
       -ov -format UDZO \
       "target/macos/Glass-${VERSION}-aarch64.dmg"

   echo "Created target/macos/Glass-${VERSION}-aarch64.dmg"
   ```
   Make it executable: `chmod +x packaging/macos/build-dmg.sh`

3. Create `packaging/linux/glass.desktop`:
   ```ini
   [Desktop Entry]
   Name=Glass Terminal
   Comment=GPU-accelerated terminal emulator with command structure awareness
   Exec=glass
   Icon=utilities-terminal
   Terminal=true
   Type=Application
   Categories=System;TerminalEmulator;
   Keywords=terminal;console;shell;
   ```

4. Add `[package.metadata.deb]` section to the root Cargo.toml (after [dev-dependencies]):
   ```toml
   [package.metadata.deb]
   maintainer = "Glass Contributors"
   copyright = "2026, Glass Contributors"
   license-file = ["LICENSE", "0"]
   depends = "$auto"
   extended-description = """\
   Glass is a GPU-accelerated terminal emulator that understands command structure. \
   It renders each command's output as a visually distinct block with exit code, \
   duration, and status information."""
   section = "x11"
   priority = "optional"
   assets = [
       ["target/release/glass", "usr/bin/", "755"],
       ["packaging/linux/glass.desktop", "usr/share/applications/", "644"],
   ]
   ```

5. Verify the Cargo.toml is still valid: `cargo metadata --format-version 1 > /dev/null`
  </action>
  <verify>
    <automated>test -f packaging/macos/Info.plist && test -f packaging/macos/build-dmg.sh && test -x packaging/macos/build-dmg.sh && test -f packaging/linux/glass.desktop && grep -q "package.metadata.deb" Cargo.toml && cargo metadata --format-version 1 > /dev/null 2>&1</automated>
  </verify>
  <done>Info.plist exists with Glass bundle identifier. build-dmg.sh is executable and references hdiutil. glass.desktop exists with Terminal=true. Cargo.toml has [package.metadata.deb] section with correct assets mapping. cargo metadata succeeds.</done>
