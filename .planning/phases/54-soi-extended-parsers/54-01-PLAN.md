  <name>Task 1: Git, Docker, kubectl parsers</name>
  <files>crates/glass_soi/src/git.rs, crates/glass_soi/src/docker.rs, crates/glass_soi/src/kubectl.rs</files>
  <behavior>
    git.rs:
    - git status output ("On branch main", "modified: file.rs", "Untracked files:") produces GitEvent records with action="status" and detail containing branch/file info
    - git diff --stat summary line ("2 files changed, 42 insertions(+), 3 deletions(-)") produces GitEvent with files_changed, insertions, deletions populated
    - git log --oneline lines ("a1b2c3d feat: add feature") produce GitEvent records with action="log" and detail containing hash + message
    - Merge conflict markers ("CONFLICT (content): Merge conflict in file.rs") produce GitEvent with action="conflict"
    - Empty or unrecognized git output falls through to freeform_parse

    docker.rs:
    - Legacy build "Step 1/5 : FROM ubuntu:22.04" produces DockerEvent with action="build-step", detail containing step info
    - BuildKit "#2 [1/3] FROM ubuntu:22.04" and "#4 ERROR: failed to solve" produce DockerEvent records
    - "Successfully built abc123" produces DockerEvent with action="built"
    - Compose "[+] Running 3/3" and container lines produce DockerEvent with action="compose-up"
    - Compose error lines produce DockerEvent with action="error"
    - Unrecognized docker output falls through to freeform

    kubectl.rs:
    - kubectl apply output ("deployment.apps/myapp configured") produces GenericDiagnostic with message containing the result
    - kubectl get pods table rows produce GenericDiagnostic per row with pod name and status in message
    - Unrecognized kubectl output falls through to freeform
  </behavior>
  <action>
Create three parser files following the exact pattern from npm.rs/jest.rs:

**git.rs (~150 lines):** Strip ANSI first. Parse line-by-line with OnceLock regex:
- `re_git_stat_summary`: `r"(\d+) files? changed(?:, (\d+) insertions?\(\+\))?(?:, (\d+) deletions?\(-\))?"` -> GitEvent with action="diff-stat"
- `re_git_log_oneline`: `r"^([0-9a-f]{7,12}) (.+)$"` -> GitEvent with action="log"
- String contains for status markers: "On branch " -> GitEvent action="status", "modified:", "new file:", "deleted:" -> GitEvent action="status" with file detail
- "CONFLICT " -> GitEvent action="conflict"
- "nothing to commit" or "working tree clean" -> GitEvent action="status" with detail="clean"
- Summary: highest severity is Info unless conflicts (Warning). One-line: "{N} records from git output".

**docker.rs (~120 lines):** Strip ANSI first. Parse:
- `re_legacy_step`: `r"^Step (\d+)/(\d+) : (.+)$"` -> DockerEvent action="build-step"
- `re_buildkit_step`: `r"^#(\d+) (.+)$"` -> DockerEvent action="build-step"
- `re_buildkit_error`: `r"^#\d+ ERROR: (.+)$"` -> DockerEvent action="error", severity Error
- "Successfully built" / "Successfully tagged" -> DockerEvent action="built"
- `re_compose_container`: `r"Container (\S+)\s+(Started|Stopped|Error|Created)"` -> DockerEvent action="compose-up" or "compose-error"
- Summary severity: Error if any error records, else Info.

**kubectl.rs (~100 lines):** Strip ANSI first. Parse:
- `re_apply_result`: `r"^(\S+) (configured|created|unchanged|deleted)$"` -> GenericDiagnostic with severity Info, message=full line
- For table rows: detect header line (NAME/STATUS/READY), then parse subsequent lines splitting on whitespace -> GenericDiagnostic per row with severity based on STATUS column (Running/Completed=Info, Error/CrashLoopBackOff=Error, Pending=Warning)
- Summary: count by severity.

All parsers: guard lines > 4096 chars with continue. Return freeform_parse if records is empty.
