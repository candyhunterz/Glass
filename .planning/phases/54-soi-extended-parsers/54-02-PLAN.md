Create four parser files following the established pattern:

**tsc.rs (~80 lines):** Simple single-regex parser. Strip ANSI first.
- `re_tsc_error`: `r"^(.+?)\((\d+),(\d+)\): (error|warning) (TS\d+): (.+)$"` -> CompilerError
- Map "error" -> Severity::Error, "warning" -> Severity::Warning
- Summary one-line: "{N} errors, {M} warnings" or "tsc: no issues found"
- Fall through to freeform if no matches

**go_build.rs (~70 lines):** Simple single-regex parser. No ANSI stripping needed (go build doesn't colorize).
- `re_go_error`: `r"^(.+?):(\d+):(\d+): (.+)$"` -> CompilerError with code=None, severity=Error
- Guard against false positives: skip lines starting with `#` (go module path comments)
- Summary: "{N} errors"

**go_test.rs (~130 lines):** More complex, handles verbose and non-verbose. No ANSI stripping (go test doesn't colorize).
- `re_run`: `r"^=== RUN\s+(\S+)"` - track current test name
- `re_result`: `r"^--- (PASS|FAIL|SKIP): (\S+) \(([0-9.]+)s\)$"` -> TestResult
- `re_ok_line`: `r"^ok\s+\S+\s+([0-9.]+)s"` -> TestSummary (package passed)
- `re_fail_line`: `r"^FAIL\s+\S+\s+([0-9.]+)s"` -> TestSummary (package failed)
- Collect indented lines after FAIL as failure message (similar to cargo_test pattern)
- If no "=== RUN" lines found, check for go build error patterns and chain to go_build::parse
- Summary: "{passed} passed, {failed} failed" or "all tests passed"

**json_lines.rs (~90 lines):** JSON parsing with serde_json::Value. No ANSI stripping.
- For each line: skip blank, skip if not starting with `{`, try `serde_json::from_str::<serde_json::Value>(line).ok()`
- Extract `level` or `severity` field -> map to Severity
- Extract `msg` or `message` field -> GenericDiagnostic message
- Extract `file` field if present -> GenericDiagnostic file
- If < 2 valid JSON lines parsed, fall through to freeform
- Summary: "{N} log entries ({errors} errors, {warnings} warnings)"
- Use serde_json::Value (not concrete types) per Phase 51 decision

All parsers: guard lines > 4096 chars with continue. Return freeform_parse if records empty.

```rust
// tsc: error format "file(line,col): error TS"
fn has_tsc_marker(output: &str) -> bool {
    static RE: OnceLock<Regex> = OnceLock::new();
    let re = RE.get_or_init(|| {
        Regex::new(r"\(\d+,\d+\): (?:error|warning) TS\d+:").expect("tsc marker regex")
    });
    re.is_match(output)
}

// go test: "--- PASS:" or "--- FAIL:" or "=== RUN   " lines
fn has_go_test_marker(output: &str) -> bool {
    output.contains("--- PASS:") || output.contains("--- FAIL:")
        || output.contains("=== RUN   ")
}
```

```rust
if has_git_marker(output) { return OutputType::Git; }
```
