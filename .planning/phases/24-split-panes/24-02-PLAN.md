<task type="auto">
  <name>Task 2: Implement per-pane scissor-clipped rendering</name>
  <files>crates/glass_renderer/src/frame.rs, src/main.rs</files>
  <action>
    1. Add `draw_pane` method to FrameRenderer in frame.rs:
       - Signature: `pub fn draw_pane(&mut self, device, queue, encoder, view, viewport: &ViewportLayout, width, height, snapshot, blocks, is_focused: bool)`
       - For each pane: build rects with pixel offsets from viewport.x/viewport.y, set scissor_rect to viewport bounds, build text buffers with TextBounds matching viewport pixel region
       - Use `pass.set_scissor_rect(viewport.x, viewport.y, viewport.width, viewport.height)` to clip all rendering
       - TextBounds: `left: viewport.x as i32, top: viewport.y as i32, right: (viewport.x + viewport.width) as i32, bottom: (viewport.y + viewport.height) as i32`
       - If is_focused and there are multiple panes, draw a 1px accent border (rgba 100, 149, 237, 255 -- cornflower blue) around the viewport rect
       - The grid_renderer.build_rects and build_text_buffers must offset cell positions by viewport.x and viewport.y
       - NOTE: This is complex. The existing GridRenderer computes positions from (0,0). For split panes, pass viewport offset so cells render at the correct absolute position. The simplest approach: add x_offset/y_offset parameters to GridRenderer::build_rects and build_text_areas, defaulting to 0 for backward compatibility.

    2. Add divider rendering:
       - After all panes render, draw 2px divider rects in the gaps between pane viewports
       - Divider color: rgba(80, 80, 80, 255) -- subtle gray
       - Dividers are drawn in a global pass (not clipped to any pane) per research recommendation
       - Compute divider positions from the gap between adjacent pane rects

    3. Update main.rs render loop (RedrawRequested):
       - Import ViewportLayout from glass_mux
       - Before rendering, compute pane layouts: `let pane_layouts = tab_root.compute_layout(&container_viewport)` where container_viewport accounts for tab bar (top) and status bar (bottom)
       - Container viewport: `ViewportLayout { x: 0, y: cell_h as u32, width: sc.width, height: sc.height - (cell_h as u32) * 2 }` (minus tab bar top + status bar bottom)
       - Pre-extract PaneRenderData for all panes: for each (session_id, viewport) in pane_layouts, lock term, snapshot, collect blocks, into a Vec
       - For single pane (pane_layouts.len() == 1): fall through to existing draw_frame call for maximum backward compatibility
       - For multiple panes: use new draw_pane per pane, then draw dividers, then draw status bar + tab bar globally
       - Status bar and tab bar continue to render globally (not per-pane) per research recommendation

    IMPORTANT: The borrow checker pattern from research Pitfall 4 -- collect all PaneRenderData (owned snapshots, owned block vecs) BEFORE entering the render phase. Do not hold session_mux borrows during rendering.

    IMPORTANT: For the initial implementation, keep the single-pane path using the existing draw_frame to minimize regression risk. The multi-pane path only activates when there are 2+ panes.
