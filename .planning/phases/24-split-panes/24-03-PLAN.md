<task type="auto">
  <name>Task 1: Keyboard shortcuts, mouse routing, and per-pane PTY resize</name>
  <files>src/main.rs, crates/glass_mux/src/session_mux.rs</files>
  <action>
    1. Add split pane keyboard shortcuts in main.rs key handling (alongside existing tab shortcuts in the Ctrl+Shift block):
       - Ctrl+Shift+D: Horizontal split. Call create_session() to spawn new PTY, then session_mux.split_pane(SplitDirection::Horizontal, new_session). Compute new per-pane dimensions and resize all panes' PTYs.
       - Ctrl+Shift+E: Vertical split. Same as above with SplitDirection::Vertical.
       - Ctrl+Shift+W: Close focused pane. Call session_mux.close_pane(focused_session_id). If returned session, call cleanup_session(). If tab was closed (tab_count decreased), handle like existing tab close. Recompute pane layouts and resize remaining panes' PTYs.

    2. Add focus navigation shortcuts:
       - Alt+Arrow (Left/Right/Up/Down): Call active_tab_root().find_neighbor(focused_pane, direction, container_viewport). If Some(target), set tab.focused_pane = target. Request redraw.
       - Implementation: Add `set_focused_pane(&mut self, session_id: SessionId)` to SessionMux that sets the active tab's focused_pane field.

    3. Add resize shortcuts:
       - Alt+Shift+Arrow: Call resize_ratio on the active tab's SplitNode tree. The direction maps to which axis to resize: Left/Right resize Horizontal splits, Up/Down resize Vertical splits. Delta: +/- 0.05 per keypress.
       - After ratio change, recompute pane layouts and resize all panes' PTYs with new dimensions.
       - Add `resize_focused_split(&mut self, direction: SplitDirection, delta: f32)` to SessionMux that calls root.resize_ratio(focused_pane, direction, delta).

    4. Add mouse click pane focus in main.rs:
       - On CursorMoved: Track cursor position.
       - On MouseInput (left click): If multiple panes exist, compute pane_layouts. Find which pane's viewport contains the click position using pane_at_position helper (see research Pattern 3). If click is in a different pane than focused, update focused_pane and request redraw.
       - The pane_at_position function: iterate pane_layouts, check if (click_x, click_y) falls within viewport bounds.

    5. Update PTY resize to be per-pane aware:
       - Extract a helper: `resize_all_panes(session_mux, frame_renderer, window_width, window_height)` that:
         a. Computes container viewport (accounting for tab bar + status bar)
         b. Calls compute_layout on active tab's root
         c. For EACH pane: compute per-pane num_cols and num_lines from pane viewport dimensions / cell size
         d. Send PtyMsg::Resize with per-pane dimensions to each pane's PTY
         e. Lock and resize each pane's Term with per-pane dimensions
       - Call this helper after: split, close pane, window resize
       - For background tabs (non-active): continue resizing with full window dimensions (they will recompute when activated)

    6. Add SPLIT-11 test in session_mux.rs:
       - Test that closing the last pane in a tab (when tab has a single Leaf) closes the entire tab
       - Test that closing one pane in a two-pane split leaves a single-pane tab

    IMPORTANT: The existing Ctrl+Shift+T (new tab) and Ctrl+Shift+W (close tab) shortcuts need disambiguation:
    - Ctrl+Shift+T: Always creates new tab (unchanged)
    - Ctrl+Shift+W: If multiple panes, close focused pane. If single pane, close tab (existing behavior).

    IMPORTANT: When computing per-pane PTY dimensions (SPLIT-09), use pane viewport width/height divided by cell dimensions, NOT the full window. This is research Pitfall 2.
  </action>
