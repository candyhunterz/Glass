<feature>
  <name>SplitTree Layout Engine</name>
  <files>crates/glass_mux/src/split_tree.rs, crates/glass_mux/src/layout.rs</files>
  <behavior>
    SPLIT-01 - Tree construction:
    - SplitNode::Leaf(id) creates a leaf node
    - SplitNode::Split { direction, left, right, ratio } creates a split node
    - Nested splits (3+ levels) work correctly

    SPLIT-02 - compute_layout:
    - Leaf node returns vec![(session_id, container)] (full container rect)
    - Horizontal split at 0.5 returns two rects splitting width evenly minus 2px gap
    - Vertical split at 0.5 returns two rects splitting height evenly minus 2px gap
    - Nested splits compute correct sub-rects recursively
    - All leaf rects have non-zero width and height
    - Sum of leaf widths + gaps equals container width (horizontal)
    - Sum of leaf heights + gaps equals container height (vertical)

    SPLIT-03 - Horizontal split gap accounting:
    - Container 1000px wide, ratio 0.5 -> left 499px, right 499px, 2px gap
    - Container 100px wide, ratio 0.3 -> left ~28px, right ~70px, 2px gap
    - Left rect x = container.x, right rect x = container.x + left_width + 2

    SPLIT-04 - Vertical split gap accounting:
    - Container 800px tall, ratio 0.5 -> top 399px, bottom 399px, 2px gap
    - Top rect y = container.y, bottom rect y = container.y + top_height + 2

    SPLIT-05 - remove_leaf:
    - Removing the only leaf returns None
    - Removing left child of Split returns right child (collapses parent)
    - Removing right child of Split returns left child (collapses parent)
    - Removing nested leaf collapses only its immediate parent
    - Removing non-existent ID returns tree unchanged

    SPLIT-06 - find_neighbor:
    - In horizontal split, Left pane has Right neighbor, Right pane has Left neighbor
    - In vertical split, Top pane has Down neighbor, Bottom pane has Up neighbor
    - Asking for non-existent direction returns None
    - Works across nested splits (finds closest pane in given direction by Manhattan distance)

    SPLIT-07 - resize_ratio:
    - Adjusting ratio by +0.05 on a Split changes its ratio
    - Ratio clamps to minimum 0.1 and maximum 0.9
    - Adjusting ratio on a Leaf is a no-op
    - Resizing a specific split (by focused pane) finds the correct ancestor
  </behavior>
  <implementation>

       - compute_layout(&self, container: &ViewportLayout) -> Vec<(SessionId, ViewportLayout)>

       - resize_ratio(&mut self, focused: SessionId, direction: SplitDirection, delta: f32) -- finds the nearest ancestor Split in the given direction and adjusts its ratio
       - leaf_count(&self) -> usize -- utility for counting panes
       - contains(&self, id: SessionId) -> bool -- utility for membership check
