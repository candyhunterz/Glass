  <action>
    **1. RectRenderer** (`crates/glass_renderer/src/rect_renderer.rs`):

    Create a wgpu instanced-rendering pipeline for colored rectangles (cell backgrounds, cursor, selection).

    - `RectInstance` struct (repr(C), Pod, Zeroable): `pos: [f32; 4]` (x, y, w, h in pixels), `color: [f32; 4]` (RGBA normalized). This is the instance buffer layout.
    - `RectRenderer` struct: holds the wgpu `RenderPipeline`, `BindGroup` for a uniform buffer (viewport resolution), and a dynamic `wgpu::Buffer` for instances.
    - `RectRenderer::new(device, surface_format)`: Create a simple vertex+fragment shader inline via `wgpu::ShaderModuleDescriptor` with WGSL source. The vertex shader takes a fullscreen quad (2 triangles from vertex_index) and scales/positions it using instance data + viewport uniform. The fragment shader outputs the instance color.
    - `RectRenderer::prepare(&mut self, device, queue, instances: &[RectInstance], width: u32, height: u32)`: Upload instance data and viewport resolution to GPU buffers.
    - `RectRenderer::render<'a>(&'a self, render_pass: &mut wgpu::RenderPass<'a>, instance_count: u32)`: Set pipeline, bind group, vertex buffer, draw instanced.

    WGSL shader approach:
    ```wgsl
    // Vertex: generate quad corners from vertex_index (0-5 for 2 triangles)
    // Instance: position (x,y,w,h) and color from instance buffer
    // Uniform: viewport width/height for pixel-to-NDC conversion
    ```

    - Vertex buffer layout: Use instance-rate step mode for RectInstance fields.
    - No index buffer needed — use 6 vertices per instance (2 triangles).

    **2. GridRenderer** (`crates/glass_renderer/src/grid_renderer.rs`):

    Convert GridSnapshot cells into glyphon TextAreas and RectInstances.

    - `GridRenderer` struct: holds `cell_width: f32`, `cell_height: f32`, `font_size: f32`, `line_height: f32`, `scale_factor: f32`.
    - `GridRenderer::new(font_system: &mut FontSystem, font_family: &str, font_size: f32, scale_factor: f32) -> Self`: Compute cell_width and cell_height from font metrics. Use `cosmic_text::Metrics::new(font_size * scale_factor, line_height * scale_factor)`. For cell_width, create a temporary Buffer, set text to "M", shape it, and measure the advance width. Store physical pixel dimensions.
    - `GridRenderer::cell_size(&self) -> (f32, f32)`: Returns (cell_width, cell_height) for resize calculations.
    - `GridRenderer::build_rects(&self, snapshot: &GridSnapshot, default_bg: Rgb) -> Vec<RectInstance>`: Walk cells, create a RectInstance for each cell whose bg differs from default_bg. Also create cursor rect based on `snapshot.cursor` shape:
      - Block: full cell rect with cursor color
      - Beam: thin (2px wide) rect at left edge of cursor cell
      - Underline: thin (2px tall) rect at bottom of cursor cell
      - Hidden: no rect
    - `GridRenderer::build_text_areas(...)`: This is the complex method. For each terminal line (0..screen_lines):
      - Collect all cells on that line in column order
      - Skip WIDE_CHAR_SPACER cells (do not add them as characters)
      - Build a `cosmic_text::Buffer` per line using `Buffer::new(font_system, metrics)`
      - Call `buffer.set_size(font_system, Some(viewport_width), Some(line_height))`
      - Use `buffer.set_rich_text()` with per-character `Attrs`:
        - `Attrs::new().family(Family::Name(font_family)).color(GlyphonColor::rgba(fg.r, fg.g, fg.b, 255))`
        - If BOLD: `.weight(Weight::BOLD)`
        - If ITALIC: `.style(Style::Italic)`
      - Call `buffer.shape_until_scroll(font_system, false)`
      - Create `TextArea` with `left: 0.0`, `top: line_idx * cell_height`, `scale: scale_factor`, appropriate TextBounds, default_color white.
      - NOTE: The Buffers need to outlive the TextArea references. Store Buffers in a Vec owned by the caller, and return TextAreas that borrow from them. Alternatively, return a struct that owns both.

    **Design note on Buffer ownership:** Since `TextArea` borrows `&Buffer`, the executor must ensure Buffers live long enough. Recommended approach: create a `FrameTextData` struct that owns `Vec<Buffer>` and produces `Vec<TextArea<'_>>` via a method that borrows from itself. Or use a simpler approach: pass `&mut Vec<Buffer>` into `build_text_areas` and return `Vec<TextArea<'_>>` borrowing from it.

    **3. FrameRenderer** (`crates/glass_renderer/src/frame.rs`):

    Orchestrate the full rendering pipeline.

    - `FrameRenderer` struct: holds `GlyphCache`, `GridRenderer`, `RectRenderer`, `default_bg: Rgb`.
    - `FrameRenderer::new(device, queue, surface_format, font_family, font_size, scale_factor) -> Self`: Initialize GlyphCache, GridRenderer, RectRenderer.
    - `FrameRenderer::draw_frame(&mut self, device, queue, view: &wgpu::TextureView, surface_config: &wgpu::SurfaceConfiguration, snapshot: &GridSnapshot)`:
      1. Build rect instances from snapshot via `grid_renderer.build_rects()`
      2. Prepare rect renderer with instances
      3. Build text Buffers and TextAreas from snapshot via `grid_renderer.build_text_areas()`
      4. Update viewport resolution
      5. Call `text_renderer.prepare()` with text areas
      6. Begin render pass with clear color matching default_bg
      7. Call `rect_renderer.render()` (backgrounds first)
      8. Call `text_renderer.render()` (text on top)
      9. End render pass, submit, (caller presents)
    - `FrameRenderer::resize(&mut self, font_system, width, height, scale_factor)`: Recompute grid dimensions if needed.
    - `FrameRenderer::cell_size(&self) -> (f32, f32)`: Delegate to GridRenderer.

    **4. Update surface.rs:**

    Make GlassRenderer fields `pub(crate)` instead of private so FrameRenderer can access device/queue/surface/surface_config. Or better: add accessor methods:
    - `pub fn device(&self) -> &wgpu::Device`
    - `pub fn queue(&self) -> &wgpu::Queue`
    - `pub fn surface_format(&self) -> wgpu::TextureFormat`
    - `pub fn surface_config(&self) -> &wgpu::SurfaceConfiguration`

    Also update `draw()` to accept an optional `&mut FrameRenderer` and `&GridSnapshot` — or better, replace the clear-only `draw()` with a method that delegates to FrameRenderer when available, falling back to clear-only if no snapshot.

    **5. Update lib.rs:**

    Add `pub mod rect_renderer; pub mod grid_renderer; pub mod frame;` and re-exports for `FrameRenderer`, `GridRenderer`, `RectRenderer`.

       - Get surface texture, create view
       - Call `frame_renderer.draw_frame(device, queue, &view, surface_config, &snapshot)`
       - Present the frame
       - Call `glyph_cache.trim()` via frame_renderer after present

    4. **In Resized:**
       - Compute new num_cols and num_lines from `(width / cell_width)` and `(height / cell_height)` using font metrics from FrameRenderer
       - Send `PtyMsg::Resize(WindowSize { num_lines, num_cols, cell_width, cell_height })` with real cell dimensions
       - Also call `term.lock().resize(TermSize::new(num_cols, num_lines))` to update the grid (CORE-07)
       - Remove the hardcoded 8x16 cell size

    5. **Imports:**
       Add imports for `glass_terminal::{snapshot_term, DefaultColors, GridSnapshot}`, `glass_renderer::{FrameRenderer}`.

    CRITICAL: The Term lock in RedrawRequested must be held only for the `snapshot_term()` call, then released immediately. The GPU draw call (`draw_frame`) must NOT hold the lock. This is the lock-minimizing GridSnapshot pattern from research.

    CRITICAL for resize: After computing new cell dimensions, also resize the `Term` itself with `term.lock().resize(TermSize)` so the grid reflows. The PTY resize message tells ConPTY the new size, and `term.resize()` tells alacritty_terminal to reflow the grid. Both are needed for CORE-07.
