  <name>Task 1: Add reorder_tab() to SessionMux with tests</name>
  <files>crates/glass_mux/src/session_mux.rs</files>
  <behavior>
    - reorder_tab(0, 2) with 3 tabs [A, B, C] produces [B, C, A]
    - reorder_tab(2, 0) with 3 tabs [A, B, C] produces [C, A, B]
    - reorder_tab(1, 1) is a no-op
    - When active_tab is the moved tab, active_tab follows it to new position
    - When active_tab is between from and to, active_tab shifts correctly
    - Out-of-bounds from or to is a no-op (early return)
    - reorder_tab with only 1 tab is a no-op
  </behavior>
  <action>
Add a `pub fn reorder_tab(&mut self, from: usize, to: usize)` method to `SessionMux`.

Implementation:
1. Guard: if `from >= self.tabs.len() || to >= self.tabs.len() || from == to` return early.
2. Remove the tab at `from` with `self.tabs.remove(from)`.
3. Insert at `to` with `self.tabs.insert(to, tab)`.
4. Adjust `self.active_tab`:
   - If `self.active_tab == from`, set `self.active_tab = to` (active tab moved).
   - Else if `from < self.active_tab && to >= self.active_tab`, decrement `self.active_tab` (tab removed before active, inserted at/after).
   - Else if `from > self.active_tab && to <= self.active_tab`, increment `self.active_tab` (tab inserted at/before active).

Add tests in the existing `#[cfg(test)] mod tests` block (at line 272). The test module already has helper infrastructure for creating mock tabs -- follow the existing pattern.

Note on `to` index semantics: `to` is the FINAL position index (post-removal), NOT an insertion-before index. This means `remove(from)` then `insert(to, tab)` is the correct sequence. The event handler in Plan 02 will convert the drop slot index to this final-position index.
  </action>

  </verify>
  <done>reorder_tab method exists with 5+ unit tests covering: basic forward move, basic backward move, no-op same index, active_tab follows moved tab, active_tab adjusts when between from/to, out-of-bounds guard.</done>

const DRAG_INDICATOR_WIDTH: f32 = 2.0;

3. Modify `build_tab_rects` signature to accept an additional `drop_index: Option<usize>` parameter:
