  <behavior>
    - Test: Block with exit_code=1 and pipeline_stage_count=2 gets pipeline_expanded=true after CommandFinished
    - Test: Block with exit_code=0 and pipeline_stage_count=3 (>2 stages) gets pipeline_expanded=true after CommandFinished
    - Test: Block with exit_code=0 and pipeline_stage_count=2 (<=2 stages) gets pipeline_expanded=false after CommandFinished
    - Test: Block with pipeline_stage_count=None (non-pipeline) gets pipeline_expanded=false
    - Test: pipeline_stage_commands stores parsed command text per stage
    - Test: expanded_stage_index defaults to None
    - Test: toggle_pipeline_expanded flips pipeline_expanded
    - Test: set_expanded_stage sets/clears expanded_stage_index
  </behavior>
  <action>
Add three new fields to Block struct:
- `pub pipeline_expanded: bool` -- overall expand/collapse state, default false
- `pub pipeline_stage_commands: Vec<String>` -- per-stage command text (parallel to pipeline_stages)
- `pub expanded_stage_index: Option<usize>` -- which single stage shows full output

Add to Block impl:
- `pub fn toggle_pipeline_expanded(&mut self)` -- flips pipeline_expanded, clears expanded_stage_index when collapsing
- `pub fn set_expanded_stage(&mut self, index: Option<usize>)` -- sets expanded_stage_index
- `pub fn pipeline_row_count(&self) -> usize` -- returns number of overlay rows this pipeline uses (0 if collapsed or non-pipeline, stage_count if expanded, stage_count + expanded_output_lines if a stage is expanded)

In handle_event for CommandFinished: after setting exit_code, add auto-expand logic:
```rust
if block.pipeline_stage_count.unwrap_or(0) > 0 {
    let stage_count = block.pipeline_stage_count.unwrap_or(0);
    let failed = block.exit_code.map_or(false, |c| c != 0);
    block.pipeline_expanded = failed || stage_count > 2;
}
```

Do NOT add command text parsing here -- that happens in main.rs at CommandExecuted time (already has the pattern with pending_command_text). The pipeline_stage_commands field will be populated externally.

Update Block::new() to initialize: pipeline_expanded: false, pipeline_stage_commands: Vec::new(), expanded_stage_index: None.
  </action>

  </verify>

3. `pub fn build_pipeline_rects(&self, blocks: &[&Block], display_offset: usize, screen_lines: usize, viewport_width: f32) -> Vec<RectInstance>` -- For each block where pipeline_expanded is true and pipeline_stages is non-empty: for each stage, emit a subtle background rect (rgba 30/30/40/0.8) one cell tall, positioned at (block separator Y + (stage_index + 1) * cell_height).
