  <behavior>
    RED phase -- write tests for these behaviors FIRST:

    1. test_undo_latest_no_snapshots: undo_latest returns Ok(None) when no snapshots exist
    2. test_undo_latest_restores_file: Create a snapshot with a file, modify the file on disk, call undo_latest -- file contents should be restored to pre-command state, outcome is Restored
    3. test_undo_deletes_new_file: Create a snapshot where blob_hash is NULL (file didn't exist pre-command), create the file on disk, call undo_latest -- file should be deleted, outcome is Deleted
    4. test_undo_skips_already_absent: Snapshot has NULL hash, file doesn't exist on disk either -- outcome is Skipped
    5. test_conflict_detection: Create pre-exec snapshot (source=parser), create watcher snapshot (source=watcher) for same command_id with post-command hash, then modify file on disk to something different from both -- outcome is Conflict with current_hash and expected_hash
    6. test_no_conflict_when_file_unchanged: Pre-exec + watcher snapshots, file on disk still matches watcher hash -- no conflict, file is Restored
    7. test_only_parser_files_restored: Snapshot has both parser and watcher files -- only parser files get restore attempts, watcher files are ignored

    GREEN phase -- implement UndoEngine:

    ```rust
    pub struct UndoEngine<'a> {
        store: &'a SnapshotStore,
    }

    impl<'a> UndoEngine<'a> {
        pub fn new(store: &'a SnapshotStore) -> Self;
        pub fn undo_latest(&self) -> Result<Option<UndoResult>>;
        fn check_conflict(&self, file_path: &Path, command_id: i64) -> Result<bool>;
        fn restore_file(&self, file_rec: &SnapshotFileRecord, command_id: i64) -> (PathBuf, FileOutcome);
    }
    ```

    Conflict detection logic:
    - Hash current file on disk with blake3
    - Find watcher snapshot for same command_id
    - Look for watcher file record matching this path
    - If watcher hash exists and current hash differs from watcher hash -> conflict
    - If no watcher data exists for this file -> no conflict (optimistic, per research recommendation)

    For confidence in UndoResult: Use Confidence::High when snapshot has parser files. The confidence comes from the ParseResult that created the snapshot -- for V1, all snapshots returned by get_latest_parser_snapshot are parser-sourced, so confidence is High.
