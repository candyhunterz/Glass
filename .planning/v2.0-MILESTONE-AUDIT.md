### Latent Integration Gap (Medium Severity)

**INT-01: Background tab PTY exit**

`SessionMux::close_pane()` always operates on `self.active_tab` (session_mux.rs line 173). If a PTY exits in a background tab (not the active tab), `close_pane(session_id)` will attempt to remove the session from the wrong tab's SplitNode tree. The session would be leaked.

- **Impact:** Low in practice -- background tab shell exits are uncommon edge cases
- **Not a blocker:** Active-tab case (the primary use case) works correctly after Phase 25 fix
- **Fix:** Add `close_pane_in_tab(tab_index, session_id)` variant to SessionMux
